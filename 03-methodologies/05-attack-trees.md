# Attack Trees: Деревья атак

## Что такое Attack Trees?

Представь, что ты планируешь ограбление сейфа. Ты рисуешь на бумаге варианты: взломать дверь, подкупить охранника, выключить камеры, узнать код от сейфа. Для каждого варианта — подварианты: дверь можно взломать ломом, отмычками или взорвать. Камеры можно выключить щитком или обрезать провода.

Если выполнить ВСЕ шаги одной ветки — ты доберёшься до сейфа. Если выполнить ЛЮБОЙ шаг другой ветки — тоже доберёшься. Некоторые шаги требуют комбинации действий.

**Attack Tree** — это иерархическая структура, которая показывает все пути достижения цели атакующего. Корень дерева — цель. Листья — конкретные действия. Узлы между ними — логические связи.

## Зачем нужны Attack Trees?

### Ситуация 1: Множество путей атаки

HR-платформа хранит резюме. Как злоумышленник может их украсть? Через SQL-инъекцию? Через XSS? Через украденные учётки? Через insider? Через уязвимость в библиотеке?

Attack Tree показывает ВСЕ пути. Не один, а все. И помогает понять, какой путь самый вероятный, самый дешёвый, самый быстрый.

### Ситуация 2: Оценка защиты

Команда внедрила WAF, MFA, шифрование. Но защищены ли ВСЕ пути? Attack Tree показывает: WAF закрыл SQL-инъекцию, MFA закрыл украденные пароли, но insider threat всё ещё открыт.

### Ситуация 3: Приоритизация

Десять путей атаки. На защиту всех — не хватит бюджета. Attack Tree считает: какой путь самый дешёвый для атакующего? На него и направляем защиту.

## Структура дерева

### Корень

Корень — это цель атакующего. Не абстрактная "взломать систему", а конкретная "украсть базу резюме из HR-платформы".

### Узлы

Узлы — это подцели. Для кражи базы нужно: получить доступ к базе, экспортировать данные, вывести данные из системы.

### Листья

Листья — конкретные действия. "Выполнить SQL-инъекцию", "Использовать украденные учётки администратора", "Скопировать файл через обход пути".

### Типы узлов

| Тип | Символ | Логика | Пример |
|-----|--------|--------|--------|
| **AND** | ⬡ | Нужно выполнить ВСЕ дочерние узлы | Получить доступ К экспортировать данные |
| **OR** | ◇ | Достаточно выполнить ЛЮБОЙ дочерний узел | SQL-инъекция ИЛИ stolen credentials ИЛИ insider |
| **NOT** | ¬ | Условие должно быть ЛОЖНЫМ | WAF НЕ активен |

## Математика Attack Trees

### Вероятность

Для **OR-узла**: вероятность = 1 − произведение (1 − Pᵢ) для всех дочерних узлов.

Пример: два пути с вероятностью 0.3 и 0.4.
P = 1 − (1−0.3) × (1−0.4) = 1 − 0.7 × 0.6 = 1 − 0.42 = 0.58

Для **AND-узла**: вероятность = произведение Pᵢ для всех дочерних узлов.

Пример: два шага с вероятностью 0.8 и 0.9.
P = 0.8 × 0.9 = 0.72

### Стоимость

Для **OR-узла**: стоимость = минимальная стоимость среди дочерних. Атакующий выбирает самый дешёвый путь.

Для **AND-узла**: стоимость = сумма стоимостей всех дочерних. Нужно выполнить все шаги.

### Время

Аналогично стоимости. OR — минимальное время, AND — сумма времени.

## Пример: Кража резюме

```
[УКРАСТЬ БАЗУ РЕЗЮМЕ]
         OR
   ______|______
  |             |
[Непосредственный]  [Через компрометацию]
   доступ к БД         HR-аккаунта
      AND                  AND
  ____|____             ____|____
 |    |    |           |         |
[SQLi] [RCE] [Backup [Phishing] [Credential
           access]              stuffing]
```

**Расчёт для "Непосредственный доступ":**

| Путь | Вероятность | Стоимость | Время |
|------|-------------|-----------|-------|
| SQL Injection | 0.3 | $100 | 4ч |
| RCE | 0.1 | $500 | 16ч |
| Backup access | 0.4 | $20 | 2ч |

P_OR = 1 − (1−0.3)(1−0.1)(1−0.4) = 1 − 0.7×0.9×0.6 = 1 − 0.378 = 0.622

C_OR = min(100, 500, 20) = $20
T_OR = min(4, 16, 2) = 2ч

**Вывод:** Самый вероятный путь — backup access (40%, $20, 2 часа).

## Attack Trees vs другие методологии

| Критерий | Attack Trees | STRIDE | Kill Chain |
|----------|--------------|--------|------------|
| Фокус | Пути достижения цели | Категории угроз | Этапы атаки |
| Количественность | Да (P, C, T) | Нет | Нет |
| Иерархичность | Да (дерево) | Нет | Да (цепочка) |
| Защита | Defense Trees | Mitigations | Countermeasures |

## Defense Trees

**Defense Tree** — зеркальное дерево, показывающее защитные меры.

К каждому листу Attack Tree можно добавить Defense Node:
- WAF блокирует SQL-инъекцию
- MFA блокирует credential stuffing
- DLP блокирует эксфильтрацию

Если защита активна — вероятность атаки снижается или обнуляется.

## Ограничения

### Сложность

Для больших систем деревья становятся огромными. Десятки узлов, сотни путей. Сложно анализировать.

### Неопределённость

Вероятности и стоимости — оценки. Реальность может отличаться.

### Динамика

Дерево статично. Реальные атаки адаптируются. Защита одного пути — атакующий идёт другим.

### Ответы на чек-лист

1. **Attack Tree** — иерархическая структура, показывающая все пути достижения цели атакующего. Корень — цель, узлы — подцели, листья — конкретные действия.

2. **Типы узлов**: AND — нужно выполнить все дочерние узлы. OR — достаточно выполнить любой дочерний узел. NOT — условие должно быть ложным.

3. **Вероятность для OR**: P = 1 − произведение (1 − Pᵢ) для всех дочерних узлов. Или: P = 1 − (1−P₁)(1−P₂)...

4. **Вероятность для AND**: P = произведение Pᵢ для всех дочерних узлов. Или: P = P₁ × P₂ × ...

5. **Defense Tree** — зеркальное дерево защитных мер. К каждому узлу Attack Tree добавляется защита. Показывает, какие контрмеры защищают от каких путей.

6. **Приоритизация** через Attack Tree: считаем вероятность, стоимость, время для каждого пути. Самый вероятный/дешёвый/быстрый путь — приоритетный для защиты.

7. **Attack Tree** показывает пути достижения цели. **Kill Chain** показывает этапы атаки. Attack Tree — "как", Kill Chain — "когда".

8. **Метрики листьев**: вероятность успеха (P), стоимость атаки (C), время атаки (T), сложность (D), необходимые ресурсы.

9. **Ограничения**: сложность больших деревьев, неопределённость оценок, статичность (не учитывает адаптацию атакующего), требует экспертизы для оценки.

10. **Attack Tree дополняет STRIDE**: STRIDE находит угрозы (категории). Attack Tree показывает пути их реализации (конкретные шаги). Вместе — полная картина.
