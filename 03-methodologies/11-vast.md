# VAST: Визуальное, агил и простое моделирование угроз

## Что такое VAST?

Представь архитектора, который рисует эскиз дома на салфетке. Простые линии, понятные формы, основные элементы. За пять минут заказчик понимает: где кухня, где спальня, где окна.

А теперь представь инженера, который строит точную 3D-модель. Каждый винт, каждая труба, каждый кабель. Месяцы работы, миллионы деталей.

**VAST** — это методология, которая находится посередине. Она визуальная, быстрая и понятная. Но при этом структурированная и repeatable.

VAST расшифровывается как Visual, Agile, and Simple Threat modeling. Создана компанией IriusRisk.

## Зачем нужен VAST?

### Ситуация 1: Agile-команда

Команда работает по Scrum. Две недели на спринт. Нет времени на недельное моделирование угроз.

VAST занимает часы, а не дни. Интегрируется в спринт: threat modeling — часть планирования.

### Ситуация 2: Разработчики без security-экспертизы

Разработчики не знают STRIDE, DREAD, PASTA. Им нужно простое средство.

VAST — визуальное. Рисуешь диаграмму, а система сама предлагает угрозы. Не нужно быть экспертом.

### Ситуация 3: Масштабирование

Компания имеет сто приложений. Нужно моделировать угрозы для всех. Ручной процесс не масштабируется.

VAST автоматизирован. Построил диаграмму — получил угрозы. Быстро, repeatable, масштабируемо.

## Три принципа VAST

### Visual (Визуальный)

Рисуешь диаграмму. Не пишешь текст, не заполняешь таблицы — рисуешь. Компоненты, связи, потоки.

Визуальное мышление быстрее и понятнее. Разработчик видит систему целиком.

### Agile (Гибкий)

VAST вписывается в agile-процессы. Не отдельная фаза, а часть разработки.

- На планировании — строим модель
- Во время разработки — обновляем
- На ревью — проверяем

### Simple (Простой)

Нет сложных формул, нет недельных процессов. Простая диаграмма — простые угрозы — простые контрмеры.

## Как работает VAST

### Шаг 1: Нарисовать диаграмму

Используется стандартная нотация (например, UML или ArchiMate). Компоненты, связи, trust boundaries.

### Шаг 2: Автоматический анализ

Инструмент (например, IriusRisk) анализирует диаграмму. Применяет rules engine. Генерирует угрозы.

Пример правил:
- Если компонент — веб-приложение, доступное из интернета → SQL-инъекция, XSS, CSRF
- Если компонент — база данных → утечка данных, tampering
- Если поток — HTTP без TLS → перехват трафика

### Шаг 3: Оценка рисков

Каждая угроза оценивается автоматически. Используется DREAD или кастомная шкала.

### Шаг 4: Генерация контрмер

Для каждой угрозы — рекомендуемая контрмера. Например:
- SQL-инъекция → параметризованные запросы
- XSS → output encoding
- Утечка данных → шифрование

### Шаг 5: Отслеживание

Контрмеры становятся задачами в трекере. Разработчик видит: что нужно сделать, почему, какой риск.

## VAST vs другие методологии

| Критерий | VAST | STRIDE | PASTA |
|----------|------|--------|-------|
| Скорость | Очень быстрый | Быстрый | Медленный |
| Автоматизация | Высокая | Низкая | Низкая |
| Требует эксперта | Нет | Да | Да |
| Интеграция | Agile/DevOps | Waterfall/Design | Enterprise |
| Визуальность | Высокая | Средняя | Низкая |
| Глубина | Средняя | Высокая | Очень высокая |

## Ограничения VAST

### Зависимость от инструмента

VAST требует платформы (IriusRisk). Без инструмента — просто набор принципов.

### Поверхностность

Автоматический анализ не заменяет эксперта. Сложные угрозы могут быть упущены.

### Стандартные угрозы

Rules engine генерирует стандартные угрозы. Zero-day, специфичные для бизнеса — не найдёт.

### Ответы на чек-лист

1. **VAST** — Visual, Agile, and Simple Threat modeling. Методология моделирования угроз, ориентированная на визуализацию, гибкость и простоту.

2. **Три принципа**: Visual (визуальное моделирование диаграммами), Agile (интеграция в спринты), Simple (простота без сложных формул).

3. **Автоматический анализ**: инструмент анализирует диаграмму, применяет rules engine, генерирует угрозы на основе типов компонентов и связей.

4. **Интеграция в agile**: threat modeling — часть планирования спринта. Обновляется во время разработки. Проверяется на ревью.

5. **Rules engine** — набор правил: если компонент типа X, доступный из Y, то угрозы Z. Например: веб-приложение в интернете → SQLi, XSS, CSRF.

6. **VAST** — быстрый, автоматизированный, для разработчиков. **STRIDE** — ручной, требует эксперта, более глубокий.

7. **Инструменты**: IriusRisk (основной), Threat Dragon (бесплатный аналог), Microsoft Threat Modeling Tool.

8. **Ограничения**: зависимость от инструмента, поверхностность (не находит сложные угрозы), стандартные угрозы (не находит zero-day).

9. **Использовать VAST** при: agile-разработке, отсутствии security-эксперта, масштабировании на множество приложений, быстрых проверках.

10. **Генерация контрмер**: для каждой угрозы rules engine предлагает стандартную контрмеру. SQLi → параметризованные запросы. Контрмеры становятся задачами в трекере.
