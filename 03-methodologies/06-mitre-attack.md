# MITRE ATT&CK: Матрица тактик и техник

## Что такое MITRE ATT&CK?

Представь библиотеку преступлений. Не вымышленных, а реальных. Каждое преступление разобрано по косточкам: как готовился, какие инструменты использовал, какие улики оставил, как его поймали.

Детективы изучают эту библиотеку. Они знают: если преступник использовал технику X — значит, нужно искать след Y. Если была техника A — значит, возможны техники B и C.

**MITRE ATT&CK** — это база знаний о реальных кибератаках. ATT&CK расшифровывается как Adversarial Tactics, Techniques, and Common Knowledge. Это не теория. Это факты, собранные из анализа тысяч реальных инцидентов.

MITRE — некоммерческая организация, которая ведёт эту базу с 2013 года. Она бесплатна, открыта и постоянно обновляется.

## Зачем нужен ATT&CK?

### Ситуация 1: Проверка защиты

Компания купила SIEM, EDR, firewall. "Мы защищены". Но от чего именно? Какие техники атакующих мы можем обнаружить? Какие — нет?

ATT&CK показывает матрицу. Команда отмечает: "Мы обнаруживаем техники T1059 (Command-Line Interface) через EDR. Не обнаруживаем T1071 (Application Layer Protocol) — нет анализа трафика". Понятно, куда вкладывать.

### Ситуация 2: Инцидент-респонс

Атака произошла. Аналитик видит в логах выполнение PowerShell. Он открывает ATT&CK: T1059.001 — PowerShell. Смотрит, какие техники обычно используются вместе с PowerShell. Находит связи: часто используется с T1003 (Credential Dumping) и T1021 (Remote Services). Проверяет эти направления.

### Ситуация 3: Планирование защиты

Компания хочет понять, какие APT-группы могут её атаковать. ATT&CK содержит профили групп. Группа APT28 использует техники X, Y, Z. Проверяем: обнаруживаем ли мы X, Y, Z? Если нет — усиливаем защиту.

## Структура ATT&CK

### Тактики

Тактика — это цель, которую преследует атакующий. Почему он делает то, что делает.

| Тактика | Описание | Пример техник |
|---------|----------|---------------|
| Initial Access | Первоначальный доступ | Phishing, Exploit Public-Facing App |
| Execution | Выполнение кода | PowerShell, Command Shell |
| Persistence | Закрепление | Registry Run Keys, Scheduled Task |
| Privilege Escalation | Повышение прав | Bypass UAC, Sudo Caching |
| Defense Evasion | Обход защиты | Masquerading, Process Injection |
| Credential Access | Доступ к учёткам | Mimikatz, Credential Dumping |
| Discovery | Разведка | System Information Discovery |
| Lateral Movement | Движение вбок | Remote Desktop, SMB/Windows Admin Shares |
| Collection | Сбор данных | Clipboard Data, Screen Capture |
| Command and Control | Управление | DNS, HTTPS, Custom Protocol |
| Exfiltration | Вывод данных | C2 Channel, Cloud Storage |
| Impact | Воздействие | Data Encrypted for Impact |

### Техники

Техника — это конкретный способ достижения тактики. Для тактики "Execution" — техники: PowerShell, Python, WMI, Scheduled Task.

Каждая техника имеет:
- ID (например, T1059.001)
- Описание
- Примеры из реальных атак
- Методы детектирования
- Методы митигации
- Связи с другими техниками

### Подтехники

Подтехника — уточнение техники. T1059 — Command and Scripting Interpreter. T1059.001 — PowerShell. T1059.003 — Windows Command Shell.

### Группы

ATT&CK содержит профили APT-групп. Какие техники использует каждая группа.

Примеры групп:
- APT28 (Fancy Bear) — российская группа
- APT29 (Cozy Bear) — российская группа
- Lazarus Group — северокорейская группа
- Carbanak — финансовые атаки

### Программы

Программы — это инструменты, используемые атакующими. Mimikatz, Cobalt Strike, PsExec. Для каждого программы — какие техники он реализует.

## Как использовать ATT&CK

### Детектирование

Для каждой техники ATT&CK предлагает методы детектирования:
- Анализ логов
- EDR-правила
- Сетевой мониторинг
- Поведенческий анализ

Команда создаёт правила детектирования для техник, релевантных их инфраструктуре.

### Митигация

Для каждой техники — контрмеры:
- Конфигурация системы
- Инструменты защиты
- Процедуры

### Оценка зрелости

Команда оценивает: сколько техник мы можем обнаружить? Сколько — митигировать? Это показывает уровень зрелости защиты.

| Уровень | Процент покрытия |
|---------|-----------------|
| Начальный | < 25% |
| Развивающийся | 25-50% |
| Определённый | 50-75% |
| Управляемый | 75-90% |
| Оптимизирующий | > 90% |

## ATT&CK vs Kill Chain

| Критерий | ATT&CK | Kill Chain |
|----------|--------|------------|
| Детализация | Очень высокая | Высокоуровневая |
| Фокус | Техники | Этапы |
| Обновление | Постоянное | Статичное |
| Использование | Детектирование, митигация | Общее понимание |
| Покрытие | Enterprise, Mobile, ICS | Общая модель |

ATT&CK — это расширение и детализация Kill Chain. Kill Chain показывает этапы, ATT&CK показывает конкретные техники внутри каждого этапа.

## Фреймворки на основе ATT&CK

### MITRE D3FEND

D3FEND — противоположность ATT&CK. Если ATT&CK описывает техники атаки, D3FEND описывает техники защиты.

| D3FEND | Что делает |
|--------|-----------|
| Harden | Укрепление системы |
| Detect | Обнаружение |
| Isolate | Изоляция |
| Deceive | Обман атакующего |
| Evict | Удаление атакующего |

### ATT&CK Navigator

Веб-инструмент для визуализации матрицы ATT&CK. Можно отмечать: какие техники обнаруживаем, какие митигаем, какие релевантны.

### CALDERA

Платформа для автоматизации adversary emulation. Позволяет эмулировать техники ATT&CK для тестирования защиты.

## Ограничения ATT&CK

### Фокус на известном

ATT&CK содержит известные техники. Zero-day атаки, новые методы — появляются с задержкой.

### Сложность

Матрица огромная. Enterprise ATT&CK содержит 14 тактик, 200+ техник, 500+ подтехник. Сложно охватить всё.

### Не стандарт

ATT&CK — база знаний, не стандарт. Каждая компания адаптирует под себя.

### Ответы на чек-лист

1. **ATT&CK** — Adversarial Tactics, Techniques, and Common Knowledge. База знаний о реальных кибератаках, собранная MITRE.

2. **Enterprise ATT&CK** содержит 14 тактик: Initial Access, Execution, Persistence, Privilege Escalation, Defense Evasion, Credential Access, Discovery, Lateral Movement, Collection, Command and Control, Exfiltration, Impact.

3. **Тактика** — цель атакующего (почему). **Техника** — конкретный способ (как). Тактика "Execution" достигается техниками: PowerShell, WMI, Command Shell.

4. **Детектирование** через ATT&CK: для каждой техники предложены методы детектирования (логи, EDR, сеть). Команда создаёт правила для техник, релевантных инфраструктуре.

5. **MITRE D3FEND** — противоположность ATT&CK. Описывает техники защиты: Harden, Detect, Isolate, Deceive, Evict. Каждая защитная техника соответствует атакующим техникам ATT&CK.

6. **ATT&CK Navigator** — веб-инструмент для визуализации матрицы. Позволяет отмечать: какие техники обнаруживаем, митигаем, какие релевантны. Цветная матрица показывает покрытие.

7. **ATT&CK** — детальная матрица техник. **Kill Chain** — высокоуровневая модель этапов. ATT&CK детализирует каждый этап Kill Chain конкретными техниками.

8. **Adversary emulation** — имитация действий реального атакующего. Используются техники ATT&CK для тестирования защиты. CALDERA — платформа для автоматизации.

9. **Группы в ATT&CK**: APT28 (Fancy Bear), APT29 (Cozy Bear), Lazarus Group, Carbanak, FIN7, Maze Team. Каждая группа имеет профиль используемых техник.

10. **Ограничения ATT&CK**: фокус на известном (zero-day с задержкой), огромная сложность (200+ техник), не стандарт (адаптация под себя), требует ресурсов на внедрение.
