# Cyber Kill Chain: Цепочка кибератаки

## Что такое Kill Chain?

Представь ограбление банка. Грабители не врываются в дверь сразу. Сначала они изучают банк: камеры, охрану, расписание инкассации. Потом готовят инструменты: маски, оружие, машину. Потом пробираются внутрь. Потом берут деньги. Потом уезжают. Потом отмывают деньги.

Если остановить грабителей на любом из этапов — ограбления не случится. Не успели изучить — не подготовились. Не подготовились — не проникли. Не проникли — не украли.

**Cyber Kill Chain** — это семь этапов, через которые проходит кибератака. От первой разведки до достижения цели. Если защита останавливает атакующего на любом этапе — атака провалена.

Модель придумали в Lockheed Martin. Она основана на военной концепции: цепочка действий, где каждое звено зависит от предыдущего.

## Зачем нужен Kill Chain?

### Ситуация 1: Понимание защиты

Компания купила firewall, antivirus, SIEM. "Мы защищены". Но от чего именно? Какой этап атаки каждый инструмент останавливает?

Kill Chain показывает: firewall останавливает delivery. Antivirus — exploitation. SIEM — обнаруживает C2. Но что с reconnaissance? Что с weaponization? Где пробелы?

### Ситуация 2: Инцидент-респонс

Атака произошла. Нужно понять: на каком этапе нас остановили? Или на каком этапе мы пропустили?

Kill Chain помогает классифицировать инцидент. "Они проникли через фишинг, установили бэкдор, но мы обнаружили на этапе C2".

### Ситуация 3: Планирование защиты

Нужно построить защиту с нуля. Kill Chain показывает, где ставить барьеры. На каждом этапе — свой инструмент защиты.

## Семь этапов Kill Chain

### Этап 1: Reconnaissance (Разведка)

**Что делает атакующий:** Собирает информацию о цели. Изучает инфраструктуру, сотрудников, технологии.

**Как это выглядит:**
- Поиск в LinkedIn сотрудников компании
- Сканирование портов целевого сервера
- Анализ DNS-записей
- Просмотр сайта компании, job postings
- Поиск утечек данных (pastebin, dark web)
- Социальная инженерия (звонки, письма)

**Как защититься:**
- Мониторинг упоминаний компании в dark web
- Honeypot-системы (ложные сервисы для привлечения разведки)
- Обучение сотрудников социальной инженерии
- Ограничение информации в публичном доступе
- Анализ OSINT-данных

**Индикаторы:** Необычное сканирование портов, запросы к DNS, посещение страниц careers.

### Этап 2: Weaponization (Вооружение)

**Что делает атакующий:** Создаёт эксплойт, вредоносный документ, payload для атаки.

**Как это выглядит:**
- Создание документа Word с макросом
- Подготовка PDF с эксплойтом
- Создание фишингового сайта
- Написание эксплойта для уязвимости
- Упаковка payload в легитимный файл

**Как защититься:**
- Блокировка макросов по умолчанию
- Sandboxing вложений
- Анализ файлов перед открытием
- Запрет выполнения скриптов из temp
- Application whitelisting

**Индикаторы:** Появление новых файлов с макросами, подозрительные документы в email.

### Этап 3: Delivery (Доставка)

**Что делает атакующий:** Доставляет weaponized payload к жертве.

**Как это выглядит:**
- Фишинговое письмо с вложением
- Заражённый USB, брошенный на парковке
- Водяной знак на сайте (watering hole)
- Поддельное обновление ПО
- Supply chain атака (заражённая библиотека)

**Как защититься:**
- Email-фильтрация (SPF, DKIM, DMARC)
- URL-фильтрация
- Запрет USB (или только read-only)
- Проверка источника ПО
- Application control

**Индикаторы:** Подозрительные email, необычные вложения, незнакомые отправители.

### Этап 4: Exploitation (Эксплуатация)

**Что делает атакующий:** Исполняет payload на системе жертвы.

**Как это выглядит:**
- Пользователь открывает вложение → макрос запускается
- Пользователь кликает ссылку → браузер эксплуатируется
- Автоматическая эксплуатация уязвимости сервиса
- Эксплуатация через drive-by download

**Как защититься:**
- Патчи и обновления
- EMET / Windows Defender Exploit Guard
- ASLR, DEP, CFG
- Least privilege (пользователь без админа)
- Application sandboxing

**Индикаторы:** Необычные процессы, запуск скриптов, обращение к реестру.

### Этап 5: Installation (Установка)

**Что делает атакующий:** Устанавливает persistence (закрепление) в системе.

**Как это выглядит:**
- Установка бэкдора
- Создание задачи в планировщике
- Модификация реестра для автозапуска
- Установка rootkit
- Создание скрытого пользователя

**Как защититься:**
- Application whitelisting
- EDR (Endpoint Detection and Response)
- Мониторинг реестра
- Мониторинг задач планировщика
- File integrity monitoring (FIM)

**Индикаторы:** Новые задачи в планировщике, изменение реестра, новые службы.

### Этап 6: Command and Control (C2)

**Что делает атакующий:** Устанавливает канал управления с сервером.

**Как это выглядит:**
- HTTPS-соединение с C2-сервером
- DNS-туннелирование
- ICMP-туннелирование
- Социальные сети как C2 (Twitter, GitHub)
- Периодические beacon-запросы

**Как защититься:**
- DNS-мониторинг
- TLS-инспекция
- Анализ beaconing-паттернов
- Запрет исходящих соединений по умолчанию
- Threat intelligence feeds (известные C2)

**Индикаторы:** Периодические запросы к подозрительным доменам, DNS-запросы необычной длины.

### Этап 7: Actions on Objectives (Действия)

**Что делает атакующий:** Достигает своей цели. Украсть, удалить, зашифровать.

**Как это выглядит:**
- Эксфильтрация данных
- Шифрование файлов (ransomware)
- Уничтожение данных
- Модификация данных
- Шпионаж
- Sabotage

**Как защититься:**
- DLP (Data Loss Prevention)
- Шифрование данных at rest
- Backup и recovery
- Аномалия-детекция
- Network segmentation

**Индикаторы:** Необычный исходящий трафик, массовое шифрование файлов, удаление данных.

## Kill Chain в защите

### Defense in Depth по этапам

| Этап | Защита | Инструменты |
|------|--------|-------------|
| Reconnaissance | OSINT monitoring | PhishLabs, dark web monitoring |
| Weaponization | Email security | Proofpoint, Mimecast |
| Delivery | URL filtering | Cisco Umbrella, Zscaler |
| Exploitation | Endpoint protection | CrowdStrike, SentinelOne |
| Installation | EDR + FIM | Carbon Black, Tripwire |
| C2 | Network monitoring | Darktrace, Vectra |
| Actions | DLP + Backup | Symantec DLP, Veeam |

### Раннее обнаружение

Чем раньше остановить — тем лучше. Обнаружение на этапе reconnaissance даёт максимум времени на реакцию. Обнаружение на этапе actions — уже поздно.

### Kill Chain и MITRE ATT&CK

MITRE ATT&CK расширяет Kill Chain. ATT&CK описывает конкретные техники для каждого этапа. Kill Chain — высокоуровневая модель, ATT&CK — детальная матрица.

## Ограничения Kill Chain

### Линейность

Kill Chain предполагает линейное прохождение этапов. В реальности атаки могут пропускать этапы, возвращаться назад, идти параллельно.

### Фокус на malware

Изначально Kill Chain создавался для malware-based атак. Для атак без malware (insider threat, credential stuffing) — менее применим.

### Однонаправленность

Kill Chain описывает атаку извне внутрь. Для lateral movement и insider threat — нужны дополнения.

### Ответы на чек-лист

1. **Семь этапов**: Reconnaissance, Weaponization, Delivery, Exploitation, Installation, Command and Control, Actions on Objectives.

2. **Weaponization** — создание payload для атаки. Документ с макросом, PDF с эксплойтом, фишинговый сайт, подготовка инструментов.

3. **Защита от Delivery**: email-фильтрация (SPF, DKIM, DMARC), URL-фильтрация, запрет USB, проверка источника ПО, application control.

4. **Beaconing** — периодические запросы от заражённой машины к C2-серверу. "Я жив, жду команд". Индикатор: регулярные запросы к одному домену с фиксированным интервалом.

5. **Раннее обнаружение лучше**, потому что даёт больше времени на реакцию. Обнаружение на reconnaissance — можно предотвратить всё. На actions — уже произошёл ущерб.

6. **MITRE ATT&CK** детализирует Kill Chain. Для каждого этапа Kill Chain ATT&CK описывает десятки конкретных техник. Kill Chain — высокоуровневый, ATT&CK — детальный.

7. **Индикаторы Installation**: новые задачи в планировщике, изменения реестра (run keys), новые службы, новые файлы в автозагрузке, модификация системных файлов.

8. **Persistence** — закрепление в системе. Механизмы, позволяющие атакующему сохранять доступ после перезагрузки. Бэкдоры, задачи планировщика, автозапуск.

9. **Ограничения Kill Chain**: линейность (реальность сложнее), фокус на malware, однонаправленность (извне внутрь), не покрывает insider threat, не покрывает supply chain.

10. **Defense in Depth по этапам**: Reconnaissance — OSINT monitoring; Weaponization — email security; Delivery — URL filtering; Exploitation — endpoint protection; Installation — EDR + FIM; C2 — network monitoring; Actions — DLP + backup.
