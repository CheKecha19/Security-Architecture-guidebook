# Diamond Model: Алмазная модель угроз

## Что такое Diamond Model?

Представь детектива, который расследует преступление. Он не просто смотрит на оружие. Он спрашивает: кто преступник? Что он хотел? Какие инструменты использовал? Какая инфраструктура у него была?

**Diamond Model** — это фреймворк для анализа кибератак. Он рассматривает четыре точки: злоумышленник, жертва, инфраструктура, инструменты. Эти четыре точки образуют алмаз.

Модель придумали аналитики из Агентства национальной безопасности США. Она помогает понять атаку целиком, а не только технические детали.

## Зачем нужен Diamond Model?

### Ситуация 1: Расследование инцидента

Компания подверглась атаке. Нужно понять: кто это был? Что он хотел? Как это повторить?

Diamond Model структурирует расследование. Не просто нас взломали, а группа X из страны Y использовала инструмент Z для кражи данных W.

### Ситуация 2: Threat Intelligence

Команда threat intelligence хочет понять: какие группы нас атакуют? Какие у них цели? Какие инструменты?

Diamond Model создаёт профили групп. Потом можно предсказать: как они будут атаковать дальше?

### Ситуация 3: Планирование защиты

Зная профиль атакующего, можно подготовить защиту. Если группа использует фишинг — усиливаем email-защиту. Если использует zero-day — готовимся к неизвестным уязвимостям.

## Четыре точки алмаза

### 1. Adversary (Злоумышленник)

**Кто атакует:**
- APT-группа (например, APT28)
- Киберпреступник (финансовая мотивация)
- Хактивист (идеологическая мотивация)
- Insider (внутренний сотрудник)
- Nation-state (государственная поддержка)

**Что описываем:**
- Кто это (название группы, страна)
- Мотивация (деньги, шпионаж, sabotage)
- Возможности (навыки, ресурсы)
- История предыдущих атак

### 2. Victim (Жертва)

**Кто пострадал:**
- Компания (какая отрасль, размер)
- Персона (конкретный сотрудник)
- Инфраструктура (какие системы)

**Что описываем:**
- Тип жертвы (банк, hospital, government)
- Значимость (почему выбрали именно её)
- Уязвимости (что привлекло атакующего)

### 3. Infrastructure (Инфраструктура)

**Какая инфраструктура используется:**
- Домены (фишинговые сайты)
- IP-адреса (C2-серверы)
- Email-адреса
- Хостинги
- VPN, прокси

**Что описываем:**
- Какие ресурсы использовал злоумышленник
- Какие ресурсы принадлежат жертве
- Какие были скомпрометированы

### 4. Capability (Возможности / Инструменты)

**Какие инструменты использовались:**
- Malware (Trojan, RAT, ransomware)
- Exploits (zero-day, известные уязвимости)
- Инструменты (Mimikatz, Cobalt Strike)
- TTPs (Tactics, Techniques, Procedures)

**Что описываем:**
- Какие инструменты применялись
- Какие техники использовались
- Какие процедуры (как шаг за шагом)

## Связи между точками

Diamond Model не просто четыре точки. Это связи между ними:

- **Adversary → Victim**: кто кого атаковал
- **Adversary → Infrastructure**: чья инфраструктура (собственная или арендованная)
- **Adversary → Capability**: какие инструменты используются
- **Victim → Infrastructure**: какие ресурсы жертвы были задействованы
- **Infrastructure → Capability**: где размещались инструменты
- **Capability → Victim**: какие инструменты использовались против жертвы

## Применение Diamond Model

### Event analysis

Анализ одной атаки. Строим алмаз для конкретного инцидента.

### Campaign analysis

Анализ серии атак. Смотрим: одна ли группа за всем стоит? Менялись ли инструменты? Цели?

### Adversary characterization

Создание профиля злоумышленника. Что мы знаем о группе X? Какие у неё цели, инструменты, инфраструктура?

## Diamond Model vs Kill Chain

| Аспект | Diamond Model | Kill Chain |
|--------|---------------|------------|
| Фокус | Кто и зачем | Как и когда |
| Результат | Профиль атакующего | Путь атаки |
| Использование | Threat intel | Incident response |
| Гранулярность | Высокая | Средняя |
| Связи | Между сущностями | Между этапами |

## Ограничения Diamond Model

### Не для защиты

Diamond Model анализирует атаки. Не помогает спроектировать защиту. Для защиты — STRIDE, PASTA.

### Требует данных

Без информации об атаке — нечего анализировать. Нужны логи, образцы malware, индикаторы.

### Субъективность

Интерпретация зависит от аналитика. Два эксперта могут построить разные алмазы.

### Ответы на чек-лист

1. **Diamond Model** — фреймворк анализа кибератак, рассматривающий четыре точки: злоумышленник, жертва, инфраструктура, возможности.

2. **Четыре точки**: Adversary (кто атакует), Victim (кто пострадал), Infrastructure (какая инфраструктура), Capability (какие инструменты и техники).

3. **Связи**: Adversary→Victim (кто кого), Adversary→Infrastructure (чья инфраструктура), Adversary→Capability (какие инструменты), Victim→Infrastructure (ресурсы жертвы), Infrastructure→Capability (где размещены инструменты), Capability→Victim (что использовалось против жертвы).

4. **Threat intel**: Diamond Model создаёт профили групп. Понимая мотивацию, инструменты, инфраструктуру — предсказываем будущие атаки.

5. **TTPs** — Tactics, Techniques, Procedures. Как именно атакующий действует. Тактика — что делает. Техника — как. Процедура — порядок шагов.

6. **Diamond Model** фокусируется на кто и зачем. **Kill Chain** — на как и когда. Diamond Model — профиль, Kill Chain — путь.

7. **Использовать Diamond Model** при: расследовании инцидентов, threat intelligence, создании профилей групп, анализе кампаний.

8. **Campaign analysis** — анализ серии атак. Выявляем: одна ли группа, менялись ли инструменты, каковы цели.

9. **Ограничения**: не для защиты (только анализ), требует данных, субъективность интерпретации, не показывает путь атаки.

10. **Incident response**: Diamond Model структурирует расследование. Понимая четыре точки, инцидент-респондент находит связи, профилирует атакующего, предсказывает дальнейшие действия.
