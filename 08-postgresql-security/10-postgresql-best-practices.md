## Часть 2: Архитектура безопасности

### Поток данных

    Client
      |
      | SSL/TLS
      v
    [Firewall] → Только разрешённые IP
      |
      v
    [pg_hba.conf] → hostssl + scram-sha-256
      |
      v
    [Role] → Групповая роль
      |
      v
    [ACL] → GRANT/REVOKE
      |
      v
    [Table] → RLS (если нужно)
      |
      v
    [Row] → Политика RLS
      |
      v
    [Audit] → pgaudit + триггеры

### Слои защиты

| Слой | Защита | Инструмент |
|------|--------|------------|
| Сеть | Firewall, VPN | iptables, AWS Security Group |
| Транспорт | SSL/TLS | OpenSSL, сертификаты |
| Аутентификация | Пароли, сертификаты | scram-sha-256, cert |
| Авторизация | Роли, права | CREATE ROLE, GRANT |
| Данные | Шифрование, RLS | pgcrypto, RLS |
| Аудит | Логирование | pgaudit, триггеры |

## Часть 3: Конфигурация по умолчанию

### postgresql.conf

    # === Безопасность ===
    ssl = on
    ssl_min_protocol_version = 'TLSv1.2'
    ssl_ciphers = 'HIGH:!aNULL:!MD5'
    password_encryption = 'scram-sha-256'
    
    # === Аудит ===
    log_connections = on
    log_disconnections = on
    log_line_prefix = '%t [%p]: [%l-1] user=%u,db=%d,app=%a,client=%h '
    log_statement = 'ddl'
    
    # === Производительность ===
    max_connections = 100
    shared_buffers = 256MB
    effective_cache_size = 1GB
    work_mem = 4MB
    maintenance_work_mem = 64MB
    
    # === WAL ===
    wal_level = replica
    archive_mode = on
    archive_command = 'cp %p /backup/wal/%f'
    
    # === Расширения ===
    shared_preload_libraries = 'pg_stat_statements,pgaudit'

### pg_hba.conf

    # Локальные подключения
    local   all             postgres                                peer
    local   all             all                                     scram-sha-256
    
    # Удалённые только через SSL
    hostssl all             all             10.0.0.0/24             scram-sha-256
    
    # Запретить всё остальное
    host    all             all             0.0.0.0/0               reject
    
    # IPv6
    hostssl all             all             ::1/128                 scram-sha-256

## Часть 4: Операционные процедуры

### Регулярные задачи

| Период | Задача | Ответственный |
|--------|--------|---------------|
| Ежедневно | Проверка бэкапов | DBA |
| Еженедельно | Анализ медленных запросов | DBA + Разработчики |
| Еженедельно | Проверка логов безопасности | Security |
| Ежемесячно | Тестирование восстановления | DBA |
| Ежемесячно | Аудит ролей и прав | Security |
| Ежеквартально | Обзор CIS Benchmarks | Security + DBA |
| Ежеквартально | Ротация ключей | Security |
| По необходимости | Патчи безопасности | DBA |

### Реагирование на инциденты

| Инцидент | Действие |
|----------|----------|
| Подозрительная активность | Изолировать IP, проверить логи |
| Потеря данных | Активировать recovery, оповестить |
| Компрометация учётных данных | Сменить пароли, отозвать сертификаты |
| Уязвимость PostgreSQL | Применить патч, проверить эксплуатацию |

## Вывод

Лучшие практики PostgreSQL — это система:
1. **Аутентификация** — кто входит
2. **Авторизация** — что можно
3. **Шифрование** — защита данных
4. **Аудит** — что произошло
5. **Харденинг** — минимизация рисков
6. **Бэкап** — восстановление
7. **Мониторинг** — видимость

Принцип: **не доверяй никому, проверяй всё, логируй всё.**
