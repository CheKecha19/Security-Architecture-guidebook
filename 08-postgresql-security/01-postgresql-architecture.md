# Архитектура PostgreSQL и модель безопасности

## Часть 1: Что такое PostgreSQL?

Представь огромную библиотеку. Там есть каталог (где описано, что есть), стеллажи (где хранятся книги), читатели (кто берёт книги), библиотекари (кто выдаёт).

**PostgreSQL** — это именно такая «библиотека» для данных. Открытая объектно-реляционная база данных. Хранит таблицы, индексы, функции, процедуры. Обеспечивает целостность, отказоустойчивость, расширяемость.

## Зачем нужна безопасность PostgreSQL?

### Ситуация 1: Утечка персональных данных

База HR-системы содержит резюме, зарплаты, адреса. Злоумышленник получил доступ — у него данные всех сотрудников.

### Ситуация 2: SQL Injection

Приложение неправильно обрабатывает входные данные. Злоумышленник вставляет SQL-код в поисковую строку — читает, изменяет, удаляет данные.

### Ситуация 3: Privilege Escalation

Пользователь `app_user` имеет права на `SELECT` из таблицы `orders`. Он создаёт функцию с `SECURITY DEFINER` от имени владельца — получает полный доступ.

## Часть 2: Архитектура процессов

### Процессы PostgreSQL

| Процесс | Роль | Безопасность |
|---------|------|------------|
| **postmaster** | Главный процесс, принимает подключения | Слушает на порту, управляет правами |
| **backend** | Обрабатывает запросы клиентов | Выполняет с правами пользователя |
| **wal writer** | Пишет WAL (Write-Ahead Log) | Журнал транзакций, критичен для recovery |
| **background writer** | Сбрасывает dirty pages | Фоновая запись на диск |
| **autovacuum** | Очищает мёртвые tuple | Необходим для performance |
| **checkpointer** | Создаёт контрольные точки | Обеспечивает консистентность |
| **stats collector** | Собирает статистику | pg_stat_* представления |

### Поток выполнения запроса

    [Клиент]
       |
       v
    [postmaster] ← принимает подключение, аутентифицирует
       |
       v
    [backend] ← создаётся новый процесс
       |
       +---> [Parser] — разбор SQL
       +---> [Planner] — план выполнения
       +---> [Executor] — выполнение
       |
       v
    [shared_buffers] ← чтение/запись данных
       |
       v
    [WAL] ← журналирование изменений
       |
       v
    [Диск] ← постоянное хранение

## Часть 3: Хранение данных

### Структура на диске

    $PGDATA/
    ├── base/              # Базы данных
    │   ├── 1/             # template1
    │   ├── 13456/         # моя база (OID)
    │   │   ├── 1234       # таблица (filenode)
    │   │   ├── 1234_fsm   # Free Space Map
    │   │   └── 1234_vm    # Visibility Map
    ├── global/            # Глобальные таблицы
    ├── pg_wal/            # WAL (ранее pg_xlog)
    ├── pg_hba.conf        # Аутентификация
    └── postgresql.conf    # Конфигурация

### Tablespace

    -- Создать tablespace на зашифрованном диске
    CREATE TABLESPACE encrypted LOCATION '/encrypted/pgdata';
    
    -- Создать таблицу в этом tablespace
    CREATE TABLE sensitive_data (...) TABLESPACE encrypted;

## Часть 4: Системный каталог

### pg_catalog

Все метаданные хранятся в системных таблицах:

| Таблица | Описание | Безопасность |
|---------|----------|--------------|
| pg_class | Таблицы, индексы, последовательности | Видно всем, но не все колонки |
| pg_user | Пользователи | Пароли хешированы |
| pg_roles | Роли | Расширенная информация |
| pg_authid | Аутентификация | Только суперпользователи |
| pg_database | Базы данных | Права на подключение |
| pg_tablespace | Tablespaces | Права на создание |

### Информационная схема

    -- Переносимый стандартный способ получить метаданные
    SELECT * FROM information_schema.tables;
    SELECT * FROM information_schema.columns;
    SELECT * FROM information_schema.table_privileges;

## Часть 5: Расширения и безопасность

### Популярные расширения

| Расширение | Описание | Безопасность |
|------------|----------|--------------|
| pgcrypto | Криптография | Шифрование, хеширование |
| pgaudit | Аудит | Логирование SQL |
| pg_stat_statements | Статистика | Performance, security monitoring |
| postgis | Геоданные | Дополнительная поверхность |
| uuid-ossp | UUID | Уникальные идентификаторы |

### Управление расширениями

    -- Установить расширение
    CREATE EXTENSION IF NOT EXISTS pgcrypto;
    
    -- Проверить
    SELECT * FROM pg_extension;
    
    -- Удалить неиспользуемые (уменьшить поверхность атаки)
    DROP EXTENSION IF EXISTS adminpack;
    DROP EXTENSION IF EXISTS file_fdw;

## Часть 6: Потоки репликации

### Физическая репликация

    [Primary] <--- streaming ---> [Standby]
       |                          |
       | WAL                       | replay
       v                          v
    [Диск]                     [Диск]

### Логическая репликация

    [Publisher] --- publication ---> [Subscriber]
       |                                |
       | logical changes                | apply
       v                                v
    [wal_level=logical]              [Подписчик]

### Безопасность репликации

    -- Пользователь для репликации
    CREATE ROLE replicator WITH REPLICATION PASSWORD 'strong-password';
    
    -- pg_hba.conf
    hostssl replication replicator 10.0.0.0/24 scram-sha-256
