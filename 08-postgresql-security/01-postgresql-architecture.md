# Архитектура PostgreSQL

## Что такое PostgreSQL?

Представь библиотеку. В библиотеке есть каталог (база данных), полки (таблицы), книги (строки). Есть библиотекарь (процесс), который обслуживает читателей. Есть охрана (аутентификация), которая проверяет пропуска.

**PostgreSQL** — именно такая "библиотека" для данных. Хранит, организует, выдаёт. Но не книги — а таблицы, строки, индексы.

## Процессы

| Процесс | Описание |
|---------|----------|
| Postmaster | Главный процесс |
| Backend | Обработка клиента |
| WAL Writer | Запись WAL |
| Checkpointer | Контрольные точки |
| Autovacuum | Очистка |
| Logger | Логирование |

## Роли

| Тип | Описание |
|-----|----------|
| Superuser | Всё |

| Group | Коллекция ролей |

## Безопасность

| Уровень | Защита |

| Аутентификация | Пароль, LDAP, Kerberos |
