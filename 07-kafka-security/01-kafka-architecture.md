# Архитектура Kafka

## Что такое Kafka?

Представь почту. Люди пишут письма. Письма сортируются по адресатам. Почтальоны разносят. Каждый получает только свои письма. Если почтальон заболел — другой берёт его маршрут. Если писем много — нанимают больше почтальонов.

**Kafka** — это именно такая "почта" для программ. Программы пишут сообщения (produce). Другие программы читают (consume). Kafka хранит, сортирует, гарантирует доставку. Если один сервер упал — другой продолжает. Если сообщений много — добавляют серверы.

## Зачем нужна безопасность Kafka?

### Ситуация 1: Конфиденциальные данные

HR-платформа отправляет через Kafka события: "новое резюме". В событии — персональные данные. Если кто-то подключится и начнёт читать все топики — утечка.

### Ситуация 2: Подмена сообщений

Злоумышленник отправляет в Kafka: "повысить зарплату пользователю X". Система обрабатывает. Проблема.

### Ситуация 3: DoS

Бесконечный поток сообщений. Kafka падает. Система не работает.

## Компоненты Kafka

### Broker (Брокер)

| Что | Описание |
|-----|----------|
| Сервер Kafka | Хранит сообщения |
| Cluster | Несколько брокеров |
| Leader | Главный для партиции |
| Replica | Копия для отказоустойчивости |

### Topic (Топик)

| Что | Описание |
|-----|----------|
| Категория сообщений | "resumes", "payments" |
| Partition | Раздел топика |
| Offset | Позиция в партиции |
| Retention | Время хранения |

### Producer (Продюсер)

| Что | Описание |
|-----|----------|
| Отправитель | Пишет в топик |
| Key | Ключ для партиционирования |
| Partition | Куда писать |
| Ack | Подтверждение |

### Consumer (Консюмер)

| Что | Описание |
|-----|----------|
| Получатель | Читает из топика |
| Group | Группа консюмеров |
| Offset | Где остановился |
| Rebalance | Перераспределение |

## Модель безопасности

### Уровни

| Уровень | Защита |
|---------|--------|
| Сеть | TLS, firewall |
| Аутентификация | SASL, SSL, mTLS |
| Авторизация | ACL |
| Шифрование | TLS, SASL_SSL |
| Аудит | Логи |

### SASL

| Механизм | Описание |
|----------|----------|
| PLAIN | Логин/пароль (небезопасно без TLS) |
| SCRAM | Salted Challenge Response |
| GSSAPI | Kerberos |
| OAUTHBEARER | OAuth2 |

## Лучшие практики

| Практика | Описание |
|----------|----------|
| TLS | Всегда |
| SASL/SCRAM | Для аутентификации |
| ACL | На каждый топик |
| Мониторинг | JMX, логи |
| Backup | Репликация |
| Обновление | Последняя версия |

### Ответы на чек-лист

1. **Kafka** — распределённая система обмена сообщениями.

2. **Компоненты**: broker, topic, producer, consumer, partition, offset.

3. **Брокер** — сервер Kafka, хранит сообщения.

4. **Топик** — категория сообщений.

5. **Partition** — раздел топика для параллелизма.

6. **Offset** — позиция в партиции.

7. **SASL**: PLAIN, SCRAM, GSSAPI, OAUTHBEARER.

8. **Безопасность нужна**: конфиденциальность, целостность, доступность.

9. **Уровни**: сеть, аутентификация, авторизация, шифрование, аудит.

10. **Лучшие практики**: TLS, SASL/SCRAM, ACL, мониторинг, backup, обновление.
