# Лучшие практики безопасности Kafka

## Чек-лист

### Сеть

- [ ] TLS 1.3
- [ ] mTLS для высокой безопасности
- [ ] Firewall, ограничение доступа
- [ ] Не экспонировать брокеры в интернет

### Аутентификация

- [ ] SASL/SCRAM-SHA-512
- [ ] Не PLAIN без TLS
- [ ] Kerberos для enterprise
- [ ] OAuth для современных систем

### Авторизация

- [ ] ACL на каждый топик
- [ ] Least privilege
- [ ] Регулярный аудит
- [ ] Запрет по умолчанию

### Шифрование

- [ ] TLS для всего трафика
- [ ] Шифрование дисков (OS-level)
- [ ] Ротация сертификатов

### Мониторинг

- [ ] JMX метрики
- [ ] Логи авторизации
- [ ] Алерты на неудачные попытки
- [ ] SIEM

### Операции

- [ ] Регулярное обновление
- [ ] Backup
- [ ] Тестирование восстановления
- [ ] Документация

## Архитектура

`
Producer → TLS → SASL/SCRAM → Broker → ACL → Topic
Consumer → TLS → SASL/SCRAM → Broker → ACL → Topic
`

## Итог

| Уровень | Защита |
|---------|--------|
| Сеть | TLS, firewall |
| Аутентификация | SASL/SCRAM |
| Авторизация | ACL |
| Шифрование | TLS, disk encryption |
| Аудит | Логи, SIEM |
