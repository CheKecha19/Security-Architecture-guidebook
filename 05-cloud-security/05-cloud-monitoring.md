# Облачный мониторинг и SIEM

## Что такое облачный мониторинг?

Представь охрану в торговом центре. В центре камеры на каждом этаже, датчики движения, охранники на постах, пульт наблюдения. Кто-то смотрит на мониторы 24/7. Если видит что-то подозрительное — реагирует.

**Облачный мониторинг** — это такая же охрана, но для облачной инфраструктуры. Кто заходил? Когда? Что сделал? Откуда пришёл? Это нормально или подозрительно?

Мониторинг собирает данные: логи, метрики, события. Анализирует. Находит аномалии. Оповещает.

## Зачем нужен мониторинг?

### Ситуация 1: Взлом администратора

Администраторскую учётку взломали. Злоумышленник зашёл ночью из другой страны. Скачал базу данных.

Мониторинг: "Администратор зашёл в 3 ночи из Бразилии. Обычно он заходит в 9 утра из Москвы". Аномалия. Оповещение.

### Ситуация 2: Утечка через S3

Кто-то открыл S3 bucket публично. Началась утечка.

Мониторинг: "Bucket стал публичным". Оповещение. Блокировка.

### Ситуация 3: DDoS

На сайт идёт огромный трафик. Серверы падают.

Мониторинг: "Трафик вырос в 100 раз". Автоматическая защита. Оповещение.

## Что мониторить?

### Логи (Logs)

| Тип | Что записывает | Пример |
|-----|----------------|--------|
| Access logs | Кто и когда обращался | S3 access logs |
| Audit logs | Административные действия | IAM audit |
| Flow logs | Сетевой трафик | VPC Flow Logs |
| Application logs | Работа приложения | Ошибки, запросы |
| Security logs | Попытки взлома | Firewall, IDS |

### Метрики (Metrics)

| Метрика | Что показывает | Порог |
|---------|----------------|-------|
| CPU | Загрузка процессора | > 80% |
| Memory | Использование памяти | > 90% |
| Disk | Заполнение диска | > 85% |
| Network | Трафик | Аномалия |
| Latency | Время ответа | > 500ms |
| Errors | Ошибки | > 1% |

### События (Events)

| Событие | Описание |
|---------|----------|
| Создание пользователя | Кто создал, когда |
| Изменение политики | Кто поменял права |
| Удаление ресурса | Что удалили, кто |
| Вход/выход | Успешный, неуспешный |
| Изменение конфигурации | Что поменяли |

## SIEM

### Что такое SIEM?

**Security Information and Event Management** — система, которая собирает логи из всех источников, коррелирует, анализирует, оповещает.

| Функция | Описание |
|---------|----------|
| Collection | Сбор логов |
| Normalization | Приведение к единому формату |
| Correlation | Связь событий между системами |
| Detection | Обнаружение угроз |
| Alerting | Оповещение |
| Reporting | Отчёты |
| Forensics | Расследование |

### Примеры SIEM

| SIEM | Облако | Описание |
|------|--------|----------|
| Splunk | AWS, Azure, GCP | Мощный, дорогой |
| Elastic SIEM | Любое | Open source |
| Azure Sentinel | Azure | Облачный, ML |
| Google Chronicle | GCP | Облачный, скорость |
| AWS Security Hub | AWS | Централизация |
| QRadar | Любое | Enterprise |

### Ответы на чек-лист

1. **Облачный мониторинг** — сбор и анализ данных о работе облачной инфраструктуры для обнаружения проблем и угроз.

2. **Три типа**: логи (записи действий), метрики (числа), события (значимые моменты).

3. **SIEM** — Security Information and Event Management. Система сбора, корреляции и анализа логов.

4. **Функции SIEM**: сбор, нормализация, корреляция, обнаружение, оповещение, отчёты, форензика.

5. **AWS**: CloudWatch (метрики/логи), CloudTrail (аудит), GuardDuty (угрозы), Security Hub (централизация).

6. **Anomaly detection** — обнаружение отклонений от нормы: время, место, объём, действия.

7. **UEBA** — User and Entity Behavior Analytics. Анализ поведения пользователей и сущностей.

8. **Хранение логов**: 1-7 лет в зависимости от требований compliance.

9. **CloudTrail** — сервис AWS для аудита. Записывает все API-вызовы.

10. **Лучшие практики**: собирать всё, централизовать, хранить долго, защитить логи, автоматизировать, тестировать.
