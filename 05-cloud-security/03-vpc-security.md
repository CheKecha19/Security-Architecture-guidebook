# VPC и сетевая безопасность в облаке

## Что такое VPC?

Представь большой офисный комплекс. В комплексе много зданий. Каждое здание — отдельный отдел. Между зданиями — дорожки. Но не все дорожки открыты. Некоторые закрыты воротами. Некоторые — только для сотрудников. Некоторые — только для руководства.

Есть главный вход. Есть служебный. Есть парковка. Есть склад — посторонним нельзя.

**VPC** (Virtual Private Cloud) — это именно такой "офисный комплекс" в облаке. Виртуальная сеть, которую ты создаёшь. Ты решаешь: какие подсети, какие маршруты, кто имеет доступ.

VPC — это твоя изолированная сеть в публичном облаке. Никто не видит, что внутри, если ты не разрешил.

## Зачем нужна VPC?

### Ситуация 1: Изоляция сред

В облаке есть development, testing, production. Они не должны видеть друг друга.

VPC: разные подсети, разные маршруты, разные правила.

### Ситуация 2: Доступ из интернета

База данных не должна быть доступна из интернета. Только из приложения.

VPC: база в приватной подсети. Приложение в публичной. Доступ только через приложение.

### Ситуация 3: Гибридное облако

У компании есть дата-центр + облако. Нужно соединить.

VPC: VPN, Direct Connect. Безопасное соединение.

## Компоненты VPC

### Подсети (Subnets)

| Тип | Описание | Аналогия |
|-----|----------|----------|
| Публичная | Доступ из интернета | Витрина магазина |
| Приватная | Только внутри VPC | Склад |
| Изолированная | Нет маршрута вообще | Сейф |

### Таблицы маршрутов (Route Tables)

Определяют: куда идёт трафик.

| Назначение | Target | Описание |
|------------|--------|----------|
| 10.0.1.0/24 | local | Внутри VPC |
| 0.0.0.0/0 | igw-xxxx | В интернет |
| 192.168.0.0/16 | vgw-xxxx | В корпоративную сеть |

### Internet Gateway (IGW)

Шлюз в интернет. Только для публичных подсетей.

### NAT Gateway

Позволяет приватным подсетям выходить в интернет (обновления), но не входить.

### Security Groups

Виртуальный firewall на уровне инстанса.

| Параметр | Описание |
|----------|----------|
| Stateful | Ответный трафик разрешён автоматически |
| Разрешительный | Только разрешающие правила |
| Уровень | Инстанс |

Пример:
- Входящий: порт 80 из 0.0.0.0/0
- Исходящий: все порты

### NACL (Network ACL)

Firewall на уровне подсети.

| Параметр | Описание |
|----------|----------|
| Stateless | Нужны правила для обоих направлений |
| Разрешительный + запрещающий | Можно явно запретить |
| Уровень | Подсеть |

### VPC Peering

Соединение двух VPC. Без интернета. Приватно.

### VPC Endpoints

Доступ к сервисам AWS без интернета. Через приватную сеть.

| Тип | Описание |
|-----|----------|
| Gateway | S3, DynamoDB |
| Interface | Остальные сервисы |

## Архитектурные паттерны

### Трёхуровневая архитектура

```
Internet → ALB → Web Tier (публичная) → App Tier (приватная) → DB Tier (приватная)
```

| Уровень | Подсеть | Доступ |
|---------|---------|--------|
| Web | Публичная | Из интернета через ALB |
| App | Приватная | Только из Web |
| DB | Приватная | Только из App |

### Hub-and-Spoke

| VPC | Назначение |
|-----|------------|
| Hub | Shared services (AD, мониторинг) |
| Spoke 1 | Production |
| Spoke 2 | Development |

Все spoke соединены с hub через peering или Transit Gateway.

### Изоляция аккаунтов

| Аккаунт | Назначение |
|---------|------------|
| Production | Только production |
| Development | Только development |
| Shared | CI/CD, мониторинг |

## Сравнение облаков

| Компонент | AWS | Azure | GCP |
|-----------|-----|-------|-----|
| VPC | VPC | Virtual Network (VNet) | VPC |
| Подсеть | Subnet | Subnet | Subnet |
| Firewall | Security Group + NACL | NSG | Firewall Rules |
| Internet Gateway | IGW | - (встроено) | - (встроено) |
| NAT | NAT Gateway | NAT Gateway | Cloud NAT |
| Peering | VPC Peering | VNet Peering | VPC Peering |
| Transit | Transit Gateway | Virtual WAN | Cloud Router |
| Endpoints | VPC Endpoints | Private Link | Private Service Connect |

## Лучшие практики

| Практика | Описание |
|----------|----------|
| Минимум публичных подсетей | Только то, что нужно |
| Security Groups — whitelist | Только разрешающие |
| NACL — дополнительный слой | Защита подсетей |
| VPC Flow Logs | Мониторинг трафика |
| Network Firewall | Централизованный контроль |
| Endpoints для сервисов | Без интернета |

### Ответы на чек-лист

1. **VPC** — Virtual Private Cloud. Изолированная виртуальная сеть в облаке.

2. **Типы подсетей**: публичная (доступ из интернета), приватная (только внутри VPC), изолированная (нет маршрутов).

3. **Security Group** — уровень инстанса, stateful, только разрешающие. **NACL** — уровень подсети, stateless, разрешающие и запрещающие.

4. **NAT Gateway** — позволяет приватным подсетям выходить в интернет, но не входить.

5. **VPC Peering** — соединение двух VPC напрямую, без интернета.

6. **VPC Endpoints** — доступ к облачным сервисам через приватную сеть, без интернета.

7. **Трёхуровневая архитектура**: Web (публичная) → App (приватная) → DB (приватная).

8. **Hub-and-Spoke** — центральный VPC (shared services) соединён с периферийными VPC.

9. **Лучшие практики**: минимум публичных подсетей, whitelist в Security Groups, NACL как доп. слой, VPC Flow Logs, Network Firewall, endpoints.

10. **VPC (AWS)** и **VNet (Azure)** — одно и то же. Разные названия у разных провайдеров.
