# Data Loss Prevention в облаке

## Что такое DLP?

Представь банк, где работают сотрудники. У каждого — доступ к данным клиентов. Один сотрудник решает: "Я уволюсь, возьму с собой базу клиентов". Копирует на флешку. Уходит.

Как этого не допустить? Нужно следить: что сотрудник делает с данными. Копирует ли? Куда отправляет? На какие сайты загружает?

**DLP** (Data Loss Prevention) — это система, которая отслеживает, как обрабатываются чувствительные данные. Не даёт их украсть, утечь, потерять.

## Зачем нужен DLP?

### Ситуация 1: Увольнение сотрудника

Разработчик сдаёт ноутбук при увольнении. DLP проверяет: не скопировал ли он данные на личный диск, не отправил ли на личную почту.

### Ситуация 2: Случайная утечка

HR-менеджер отправил резюме на Gmail вместо корпоративный email. DLP блокирует: "Этот документ содержит ПДн, нельзя отправлять на внешние почты".

### Ситуация 3: Утечка через облако

Сотрудник загрузил конфиденциальный документ в Google Drive. DLP: "Нельзя загружать в облачные сервисы".

## Типы DLP

### Endpoint DLP

| Что контролирует | Как |
|------------------|-----|
| USB, Bluetooth | Блокировка передачи |
| Clipboard | Запрет копирования |
| Print screen | Блокировка скриншотов |
| Email | Проверка вложений |
| Upload | Контроль загрузок |

### Network DLP

| Что контролирует | Как |
|------------------|-----|
| Email | Сканирование писем |
| Web traffic | Блокировка загрузок |
| FTP | Мониторинг передач |
| Cloud apps | CASB |

### Cloud DLP

| Что контролирует | Как |
|------------------|-----|
| S3, Blob, GCS | Сканирование хранилищ |
| SaaS | Office 365, Google Workspace |
| Database | Обнаружение данных |
| API | Проверка вызовов |

## Как DLP находит данные?

### Методы обнаружения

| Метод | Описание | Пример |
|-------|----------|--------|
| Regex | Шаблоны | \d{4}-\d{4}-\d{4}-\d{4} — номер карты |
| Keywords | Ключевые слова | "паспорт", "ИНН", "снилс" |
| Fingerprinting | Уникальные отпечатки | Контракт, шаблон |
| ML | Машинное обучение | Классификация документов |
| EDM | Exact Data Matching | Точное совпадение |

### Классификация данных

| Уровень | Примеры |
|---------|---------|
| Public | Публичная информация |
| Internal | Внутренние документы |
| Confidential | Коммерческая тайна |
| Restricted | ПДн, финансы, здоровье |

## Реакция на инцидент

| Действие | Описание |
|----------|----------|
| Block | Блокировка действия |
| Allow | Разрешение |
| Quarantine | Изоляция для проверки |
| Encrypt | Шифрование |
| Alert | Оповещение |
| Log | Запись в лог |

### Ответы на чек-лист

1. **DLP** — Data Loss Prevention. Система предотвращения утечки данных.

2. **Три типа**: Endpoint (устройства), Network (сеть), Cloud (облако).

3. **Методы обнаружения**: regex, keywords, fingerprinting, ML, EDM.

4. **Fingerprinting** — создание уникальных отпечатков документов. DLP распознаёт: "Это контракт X".

5. **Уровни**: Public, Internal, Confidential, Restricted.

6. **Действия**: block, allow, quarantine, encrypt, alert, log.

7. **CASB** — Cloud Access Security Broker. Посредник между пользователем и облаком. Контролирует доступ.

8. **Облачные**: AWS Macie, Azure Purview, Google Cloud DLP.

9. **Лучшие практики**: классификация, минимизация, обнаружение, реакция, обучение, тестирование, аудит.

10. **Compliance**: GDPR, PCI DSS требуют защиты данных. DLP помогает.
