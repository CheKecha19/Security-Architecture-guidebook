# JWT: JSON Web Token

## Что такое JWT?

Представь, что у тебя есть пропуск в музей. На пропуске написано: имя, дата, уровень доступа. Но не просто написано — запечатано. Печатью. Если кто-то подделает — печать не совпадёт.

**JWT** — это именно такой "запечатанный пропуск". Структура: заголовок, данные, подпись. Всё вместе — токен. Подпись гарантирует, что никто не подделал.

## Структура

| Часть | Описание | Пример |
|-------|----------|--------|
| Header | Алгоритм и тип | {"alg":"RS256","typ":"JWT"} |
| Payload | Данные (claims) | {"sub":"123","name":"Alice"} |
| Signature | Подпись | HMAC или RSA |

## Типы JWT

| Тип | Описание | Использование |
|-----|----------|---------------|
| JWS | Подписанный | Стандарт |
| JWE | Зашифрованный | Конфиденциальность |

## Алгоритмы

| Алгоритм | Тип | Описание |
|----------|-----|----------|
| HS256 | Симметричный | HMAC + SHA-256 |
| HS512 | Симметричный | HMAC + SHA-512 |
| RS256 | Асимметричный | RSA + SHA-256 |
| RS512 | Асимметричный | RSA + SHA-512 |
| ES256 | Асимметричный | ECDSA + SHA-256 |
| EdDSA | Асимметричный | Ed25519 |

## Валидация

| Что проверять | Описание |
|---------------|----------|
| Signature | Подпись корректна? |
| exp | Не просрочен? |
| iat | Выдан не в будущем? |
| iss | Доверенный issuer? |
| aud | Для нашего приложения? |
| sub | Существует? |

## Безопасность

| Угроза | Защита |
|--------|--------|
| Подделка | Проверка подписи |
| Перехват | HTTPS, короткий срок |
| None algorithm | Запретить "alg": "none" |
| Weak secret | Длинный ключ |
| Sensitive data | Не хранить secrets в payload |
