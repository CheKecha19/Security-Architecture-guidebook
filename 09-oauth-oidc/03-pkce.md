# PKCE: Proof Key for Code Exchange

## Что такое PKCE?

Представь, что ты оставляешь машину на парковке. Ты не просто закрываешь дверь. Ты ставишь на сигнализацию. На сигнализации — код. Код знаешь только ты. Если кто-то попытается открыть — сигнализация сработает.

**PKCE** — это именно такая "сигнализация" для Authorization Code. Client генерирует секрет. Отправляет хеш секрета. Получает код. Отправляет код + секрет. Если секрет не совпадает — токен не выдаётся.

## Зачем нужен PKCE?

### Ситуация 1: Мобильное приложение

Мобильное приложение не может хранить client_secret. Любой может декомпилировать приложение и украсть секрет. PKCE не требует client_secret.

### Ситуация 2: SPA

Single Page Application — код в браузере. client_secret виден всем. PKCE решает эту проблему.

## Процесс

| Шаг | Что происходит |
|-----|----------------|
| 1 | Client генерирует code_verifier (случайная строка) |
| 2 | Client вычисляет code_challenge = SHA256(code_verifier) |
| 3 | Client отправляет code_challenge в запросе авторизации |
| 4 | Authorization Server запоминает code_challenge |
| 5 | Client получает код |
| 6 | Client отправляет код + code_verifier |
| 7 | Server проверяет SHA256(code_verifier) == code_challenge |
| 8 | Если совпадает — выдаёт токен |

## Параметры

| Параметр | Описание |
|----------|----------|
| code_challenge | SHA256(code_verifier) |
| code_challenge_method | S256 или plain |
| code_verifier | Случайная строка 43-128 символов |

## Безопасность

| Угроза | Защита |
|--------|--------|
| Перехват кода | Без code_verifier код бесполезен |
| Подмена client | code_verifier знает только настоящий client |

## Когда использовать

| Сценарий | Обязательность |
|----------|----------------|
| Мобильные | Обязательно |
| SPA | Обязательно |
| Веб (confidential) | Рекомендуется |
