# Client Credentials Flow

## Что такое Client Credentials?

Представь склад. На склад приезжает грузовик. Грузовик — не человек, а машина. У машины нет паспорта. Но у машины есть номер и пропуск. Пропуск привязан к машине. Показывает: эта машина может заезжать на склад.

**Client Credentials Flow** — это именно такой "пропуск для машины". Когда нужен доступ без пользователя. Server к server. Сервис к сервису. Машина к складу.

## Зачем нужен?

### Ситуация 1: Микросервисы

Сервис A хочет получить данные от сервиса B. Нет пользователя. Есть только сервисы. Сервис A получает токен по client_credentials. Обращается к сервису B.

### Ситуация 2: Ночной бэкап

Скрипт запускается ночью. Делает бэкап. Нет пользователя. Скрипт аутентифицируется по client_id и client_secret.

### Ситуация 3: API для партнёров

Партнёр хочет получить данные. Не через пользователя, а напрямую. Регистрируется как client. Получает client_id и client_secret.

## Процесс

| Шаг | Что происходит |
|-----|----------------|
| 1 | Client отправляет client_id + client_secret на /token |
| 2 | Authorization Server проверяет |
| 3 | Authorization Server возвращает Access Token |

## Параметры

| Параметр | Описание |
|----------|----------|
| grant_type | client_credentials |
| client_id | ID приложения |
| client_secret | Секрет приложения |
| scope | Область доступа |

## Безопасность

| Угроза | Защита |
|--------|--------|
| Утечка secret | Хранить в secrets manager |
| Перехват | HTTPS |
| Scope | Минимальные права |

## Когда использовать

| Сценарий | Почему |
|----------|--------|
| Server-to-server | Нет пользователя |
| Скрипты | Автоматизация |
| API партнёров | Прямой доступ |
| Микросервисы | Внутренние вызовы |
