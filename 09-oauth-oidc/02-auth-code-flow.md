# Authorization Code Flow

## Что такое Authorization Code Flow?

Представь, что ты хочешь получить пропуск в закрытое здание. Ты подходишь к охране. Охрана звонит в офис и говорит: "Тут человек хочет зайти, его зовут Вася". Офис отвечает: "Да, Вася у нас в списке. Вот код: 12345". Охрана даёт тебе код. Ты идёшь к турникету, вводишь код, получаешь пропуск.

**Authorization Code Flow** — это именно такой процесс. Приложение (client) хочет доступ. Оно перенаправляет пользователя к Authorization Server. Пользователь входит и соглашается. Authorization Server даёт код. Приложение обменивает код на токен.

## Почему так сложно?

Почему не дать токен сразу? Потому что код — короткий, одноразовый, привязанный к client. Если его перехватят — ничего страшного. Токен — длинный, многоразовый, опасный в чужих руках. Код можно передавать через браузер. Токен — только напрямую.

## Шаги

| Шаг | Кто | Что делает |
|-----|-----|------------|
| 1 | Client | Перенаправляет пользователя на /authorize |
| 2 | User | Входит в систему |
| 3 | User | Соглашается дать доступ |
| 4 | Authorization Server | Перенаправляет на redirect_uri с кодом |
| 5 | Client | Отправляет код + client_secret на /token |
| 6 | Authorization Server | Возвращает Access Token + Refresh Token |

## Параметры запроса

| Параметр | Описание | Пример |
|----------|----------|--------|
| response_type | Тип ответа | code |
| client_id | ID приложения | my-app |
| redirect_uri | Куда вернуть | https://app/callback |
| scope | Что разрешить | read profile |
| state | Защита от CSRF | random-string |

## Параметры обмена

| Параметр | Описание |
|----------|----------|
| grant_type | authorization_code |
| code | Код от пользователя |
| redirect_uri | Тот же URI |
| client_id | ID приложения |
| client_secret | Секрет приложения |

## Безопасность

| Угроза | Защита |
|--------|--------|
| Перехват кода | Код одноразовый, короткий |
| CSRF | Параметр state |
| Подмена redirect_uri | Регистрация URI |
| Утечка client_secret | Хранить безопасно |

## Когда использовать

| Сценарий | Почему |
|----------|--------|
| Веб-приложение | Безопасный, стандартный |
| Мобильное | С PKCE |
| SPA | С PKCE |
