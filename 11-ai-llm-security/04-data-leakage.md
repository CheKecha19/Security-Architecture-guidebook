# Data Leakage через LLM

## Что такое data leakage?

Представь библиотекаря, который прочитал все книги в библиотеке, включая личные дневники. Ты спрашиваешь: "Какая погода была вчера?" Библиотекарь отвечает. А потом добавляет: "Кстати, я прочитал в дневнике Анны, что она встречается с Петей. Вот её номер телефона".

**Data leakage** — это именно такая проблема. LLM помнит обучающие данные. Может случайно выдать конфиденциальную информацию.

## Типы утечек

| Тип | Описание | Пример |
|-----|----------|--------|
| Memorization | Запоминание | Точное воспроизведение |
| Inference | Вывод | Информация о данных |
| Membership | Принадлежность | Был ли пример в данных |

## Защита

| Метод | Описание |
|-------|----------|
| Differential privacy | Шум в данных |
| Data sanitization | Удаление sensitive |
| Output filtering | Проверка ответов |
| Rate limiting | Ограничение запросов |

## Лучшие практики

| Практика | Описание |
|----------|----------|
| Не sensitive в обучении | Не использовать |
| Проверка | Перед публикацией |
| Monitoring | На утечки |
| Retraining | Удаление утечек |
