# Архитектура LLM

## Что такое LLM?

Представь библиотеку с миллиардами книг. В библиотеке работает библиотекарь. Библиотекарь не читал все книги. Но он знает, какие слова часто встречаются вместе. Какие фразы следуют друг за другом. Если ты спрашиваешь: "Что такое любовь?" — он составляет ответ из фраз, которые встречаются рядом со словом "любовь".

**LLM (Large Language Model)** — это именно такой "библиотекарь". Не человек, а программа. Обучена на огромном количестве текста. Знает статистику слов. Не понимает смысл, но умеет предсказывать: какое слово идёт дальше.

## Трансформер

| Компонент | Описание |
|-----------|----------|
| Attention | Что важно в контексте |
| Self-attention | Связь слов между собой |
| Multi-head | Несколько уровней внимания |
| Feed-forward | Обработка |
| Layer normalization | Стабилизация |

## Токены

| Понятие | Описание |
|---------|----------|
| Токен | Часть слова или слово |
| Context window | Сколько токенов помнит |
| Embedding | Векторное представление |

## Обучение

| Этап | Описание |
|------|----------|
| Pre-training | На больших данных |
| Fine-tuning | На специфических |
| RLHF | С человеческой обратной связью |

## Угрозы

| Угроза | Описание |
|--------|----------|
| Prompt injection | Вредоносный промпт |
| Data leakage | Утечка обучающих данных |
| Jailbreaking | Обход ограничений |
| Hallucination | Вымышленные факты |

| Output filtering | Фильтрация ответов |
| Rate limiting | Ограничение запросов |
| Monitoring | Аномалии |
