# Основы безопасности Java

## Что такое Java Security?

Представь театр. В театре есть сцена, зрительный зал, кулисы, актёры, декорации. За кулисами — работают техники. В зале — сидят зрители. На сцене — актёры. Каждый знает своё место. Зритель не может выйти на сцену. Актёр не может зайти в технику без разрешения.

**Java Security** — это именно такой "театр" для программ. JVM (Java Virtual Machine) — это театральная площадка. Каждая программа — актёр. У каждого — своя роль, свои права, своё место. Одна программа не может вмешаться в работу другой. Не может читать чужие данные. Не может делать то, что ей не разрешено.

## Зачем нужна Java Security?

### Ситуация 1: Вредоносный код

Пользователь загружает Java-апплет на сайт. Апплет пытается удалить файлы на диске. Security Manager блокирует: "У тебя нет прав на удаление файлов".

### Ситуация 2: Доступ к памяти

Одно приложение пытается прочитать память другого. JVM блокирует: "Каждый живёт в своей коробке".

### Ситуация 3: Подмена класса

Злоумышленник подменяет стандартный класс Java своим. ClassLoader проверяет подпись. Подмена обнаружена.

## Архитектура безопасности Java

### Уровни защиты

| Уровень | Что защищает | Как |
|---------|--------------|-----|
| Байт-код | Исполняемый код | Верификация |
| ClassLoader | Загрузка классов | Иерархия, подпись |
| SecurityManager | Доступ к ресурсам | Policy |
| AccessController | Контроль доступа | Permissions |
| Cryptography | Шифрование | JCA, JCE |

### Верификация байт-кода

Перед выполнением JVM проверяет байт-код:
- Корректность инструкций
- Корректность стека
- Корректность типов
- Корректность переменных

### ClassLoader

| Загрузчик | Что загружает |
|-----------|---------------|
| Bootstrap | Стандартные классы (rt.jar) |
| Extension | Расширения |
| Application | Пользовательские классы |

Каждый ClassLoader имеет родителя. Иерархия защищает от подмены.

### Security Manager

Позволяет или запрещает действия:

| Действие | Permission |
|----------|------------|
| Чтение файла | FilePermission |
| Запись файла | FilePermission |
| Сетевое соединение | SocketPermission |
| Исполнение процесса | RuntimePermission |
| Рефлексия | ReflectPermission |

### Policy файл

```
grant codeBase "file:/home/user/app/" {
    permission java.io.FilePermission "/tmp/*", "read,write";
    permission java.net.SocketPermission "localhost:1024-", "connect";
};
```

## Java SE Security

### Модульная система (Java 9+)

| Модуль | Описание |
|--------|----------|
| java.base | Основа |
| java.security.jgss | Kerberos |
| java.security.sasl | SASL |
| jdk.crypto.ec | ECC |

### Permissions

| Permission | Описание |
|------------|----------|
| AllPermission | Все права |
| SecurityPermission | Права security |
| UnresolvedPermission | Не разрешённое |
| AWTPermission | Графика |
| NetPermission | Сеть |

## Лучшие практики

| Практика | Описание |
|----------|----------|
| Не отключать SecurityManager | Всегда включён |
| Минимум прав | Policy с минимумом |
| Подписывать код | JAR signing |
| Обновлять JDK | Последние патчи |
| Использовать модули | JPMS для изоляции |

### Ответы на чек-лист

1. **Java Security** — архитектура защиты JVM. Уровни: байт-код, ClassLoader, SecurityManager, AccessController.

2. **Уровни**: байт-код (верификация), ClassLoader (загрузка), SecurityManager (доступ), AccessController (контроль), Cryptography (шифрование).

3. **Верификация** проверяет корректность байт-кода перед выполнением.

4. **ClassLoader** загружает классы иерархически. Bootstrap → Extension → Application.

5. **SecurityManager** контролирует доступ к ресурсам: файлы, сеть, процессы.

6. **Policy** описывает, какие права у какого кода.

7. **Модули Java 9+** (JPMS) — изоляция компонентов.

8. **Permissions**: FilePermission, SocketPermission, RuntimePermission, ReflectPermission.

9. **Лучшие практики**: не отключать SecurityManager, минимум прав, подписывать код, обновлять JDK, использовать модули.

10. **ClassLoader важен**, потому что защищает от подмены классов.
