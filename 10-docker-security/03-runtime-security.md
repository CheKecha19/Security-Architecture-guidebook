# Безопасность runtime Docker

## Seccomp

| Что | Описание |
|-----|----------|
| Фильтр syscall | Разрешает только нужные |
| Профиль | По умолчанию ~44 syscall |
| Custom | Свой профиль |

## AppArmor

| Что | Описание |
|-----|----------|
| Профиль | Правила для процесса |
| Путь | Какие файлы доступны |
| Сеть | Какие порты |
| Capabilities | Какие привилегии |

## SELinux

| Что | Описание |
|-----|----------|
| Label | Метки для контейнеров |
| Policy | Правила доступа |
| Enforcing | Принудительное |

## Capabilities

| Capability | Описание |
|------------|----------|
| CAP_NET_ADMIN | Управление сетью |
| CAP_SYS_ADMIN | Администрирование |
| CAP_SYS_PTRACE | Отладка |
| CAP_SYS_MODULE | Модули ядра |
| CAP_DAC_READ_SEARCH | Обход проверки прав |

## Лучшие практики

| Практика | Описание |
|----------|----------|
| --cap-drop=ALL | Убрать все capabilities |
| --security-opt | Seccomp, AppArmor |
| --read-only | Только чтение |
| --tmpfs | Временная файловая система |
| --user | Не root |
| --memory | Ограничение памяти |
| --cpus | Ограничение CPU |
