# Сетевая безопасность Docker

## Типы сетей

| Тип | Описание |
|-----|----------|
| Bridge | По умолчанию |
| Host | Общий с хостом |
| None | Без сети |
| Overlay | Между хостами |
| Macvlan | Свой MAC |

## Bridge

| Параметр | Описание |
|----------|----------|
| Docker0 | Виртуальный интерфейс |
| Subnet | 172.17.0.0/16 |
| Gateway | 172.17.0.1 |
| Isolation | Между контейнерами |

## Порты

| Проблема | Решение |
|----------|---------|
| Публикация | -p только нужные |
| Binding | 127.0.0.1, не 0.0.0.0 |
| EXPOSE | Не публикует автоматически |

## Network Policies

| Параметр | Описание |
|----------|----------|
| Ingress | Входящий |
| Egress | Исходящий |
| Deny all | Запретить всё |
| Allow specific | Разрешить нужное |

## Лучшие практики

| Практика | Описание |
|----------|----------|
| Не host | Избегать --network host |
| Ограничение портов | Только необходимые |
| Сегментация | Отдельные сети |
| Firewall | iptables, ufw |
| Monitoring | Трафик |
