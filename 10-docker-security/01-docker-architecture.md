# Архитектура Docker

## Что такое Docker?

Представь корабль. На корабле есть контейнеры. Каждый контейнер — изолирован. В одном — мебель. В другом — электроника. В третьем — одежда. Контейнеры не смешиваются. Каждый имеет свой список содержимого. Корабль перевозит их все.

**Docker** — это именно такой "корабль" для программ. Контейнер содержит приложение и всё, что ему нужно. Библиотеки, настройки, зависимости. Контейнер изолирован от других. Работает одинаково везде.

## Компоненты

| Компонент | Описание | Аналогия |
|-----------|----------|----------|
| Docker Daemon | Главный процесс | Капитан корабля |
| containerd | Управление контейнерами | Первый помощник |
| runc | Запуск контейнеров | Механик |
| Docker CLI | Команды | Рация |
| Registry | Хранилище образов | Порт |

## Изоляция

| Механизм | Описание |
|----------|----------|
| Namespaces | PID, NET, IPC, MNT, UTS, USER, CGROUP |
| Cgroups | Ограничение ресурсов |
| UnionFS | Слои файловой системы |
| Capabilities | Привилегии Linux |

## Угрозы

| Угроза | Описание |
|--------|----------|
| Container escape | Выход из контейнера |
| Privileged mode | Полный доступ к хосту |
| Image vulnerabilities | Уязвимости в образе |
| Secrets in images | API-ключи в Dockerfile |
| Network exposure | Открытые порты |

## Лучшие практики

| Практика | Описание |
|----------|----------|
| Не root | Пользователь без прав |
| Read-only | Файловая система только для чтения |
| Cap drop | Убрать ненужные capabilities |
| Resource limits | Ограничение CPU, памяти |
| Scanning | Сканирование образов |
