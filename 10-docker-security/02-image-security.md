# Безопасность Docker образов

## Что такое образ?

Представь рецепт блюда. Рецепт описывает: какие ингредиенты, в каком порядке, как готовить. По рецепту можно приготовить блюдо сколько угодно раз. Каждый раз одинаково.

**Docker образ** — это именно такой "рецепт" для контейнера. Описывает: какая ОС, какие библиотеки, какое приложение. По образу создаётся контейнер. Всегда одинаковый.

## Слои

| Слой | Описание |
|------|----------|
| Base | Основа: Alpine, Ubuntu |
| Dependencies | Библиотеки |
| Application | Код |
| Config | Настройки |

## Угрозы образов

| Угроза | Описание | Пример |
|--------|----------|--------|
| Уязвимости | CVE в базовом образе | Log4j |
| Секреты | Ключи в Dockerfile | API key |
| Лишнее | Ненужные пакеты | Компиляторы |
| Malware | Вредоносный код | Поддельный образ |

## Distroless

| Характеристика | Описание |
|----------------|----------|
| Нет shell | Нет /bin/sh |
| Нет package manager | Нет apt |
| Минимум | Только приложение |
| Безопасность | Меньше поверхности атаки |

## Сканирование

| Инструмент | Описание |
|------------|----------|
| Trivy | Open source, простой |
| Snyk | Коммерческий |
| Clair | От Red Hat |

## Лучшие практики

| Практика | Описание |
