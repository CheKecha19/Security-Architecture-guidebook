# CIS Controls: 18 базовых контролей безопасности

## Что такое CIS Controls?

Представь список покупок в магазине. Не абстрактный "купи еду", а конкретный: молоко, хлеб, яйца, масло. Конкретные пункты, которые можно проверить.

**CIS Controls** — это именно такой список для безопасности. 18 конкретных мер, которые нужно внедрить. Не теории, а практика. Не "защититесь", а "включите MFA", "сделайте инвентаризацию", "патчите уязвимости".

CIS (Center for Internet Security) — некоммерческая организация. Они собирают лучшие практики от экспертов по всему миру. Обновляют регулярно.

## Зачем нужны CIS Controls?

### Ситуация 1: Не знаем, с чего начать

Компания хочет стать безопаснее. Но где начать? Что важнее?

CIS Controls даёт приоритизированный список. Начните с Implementation Group 1 (базовые). Потом Group 2. Потом Group 3.

### Ситуация 2: Быстрая проверка

Аудитор пришёл. Нужно быстро показать: "У нас есть базовая защита".

Проверяем 18 контролей. Если базовые есть — уже хорошо.

### Ситуация 3: Защита от 80% атак

CIS говорит: если внедрить первые 6 контролей (IG1), вы защититесь от 80% атак. Это не 100%, но уже много.

## Примеры конкретных действий

### Control 1: Inventory

- Ведите список всех устройств (компьютеры, серверы, телефоны, IoT)
- Используйте инструменты обнаружения (Nmap, Lansweeper)
- Удаляйте неизвестные устройства
- Обновляйте инвентарь регулярно

### Control 4: Secure Configuration

- Используйте hardened образы
- Отключайте ненужные сервисы
- Применяйте CIS Benchmarks
- Проверяйте конфигурацию

### Control 7: Vulnerability Management

- Сканируйте уязвимости еженедельно
- Приоритизируйте по CVSS
- Патчите критичные за 24-72 часа
- Проверяйте после патча

## CIS Benchmarks

Дополнение к Controls. Конкретные рекомендации для:
- Операционных систем (Windows, Linux)
- Браузеров (Chrome, Firefox)
- Облаков (AWS, Azure)
- Баз данных (PostgreSQL, MySQL)
- Контейнеров (Docker, Kubernetes)

## CIS Controls vs NIST CSF

| Критерий | CIS Controls | NIST CSF |
|----------|--------------|----------|
| Формат | Конкретный список | Функции и категории |
| Приоритизация | Да (IG1, IG2, IG3) | Нет |
| Действия | Очень конкретные | Общие |
| Быстрый старт | IG1 за 80% | Профили |
| Аудит | Легко проверить | Требует адаптации |

## Ограничения CIS Controls

### Не покрывает всё

18 контролей — база. Для полной защиты нужно больше.

### Фокус на технологиях

Меньше внимания процессам, людям, бизнес-рискам.

### Быстро устаревают

Технологии меняются. Нужно обновлять.

### Ответы на чек-лист

1. **CIS Controls** — 18 приоритизированных мер безопасности от Center for Internet Security. Конкретные действия для защиты.

2. **18 контролей** в версии 8.

3. **IG1 (базовые)**: Inventory Assets, Inventory Software, Data Protection, Secure Configuration, Account Management, Access Control.

4. **IG1 защищает от 80%** распространённых атак.

5. **CIS Benchmarks** — конкретные рекомендации по настройке систем, приложений, облаков.

6. **CIS Controls** — конкретный список действий с приоритетами. **NIST CSF** — фреймворк функций без приоритетов. CIS проще для старта.

7. **Каждый контроль содержит**: зачем важен, процедуры внедрения, показатели, зависимости.

8. **Приоритизация**: IG1 (базовые, защита 80%), IG2 (фундаментальные), IG3 (продвинутые).

9. **Использовать CIS** при: быстром старте, аудите базовой защиты, приоритизации, проверке соответствия.

10. **Ограничения**: не покрывает всё, фокус на технологиях, быстро устаревают, нужно дополнять.
