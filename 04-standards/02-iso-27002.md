# ISO 27002: Практические рекомендации по ИБ

## Что такое ISO 27002?

Представь учебник для поваров. В нём не написано "у вас должен быть ресторан". В нём написано: "вот как правильно мыть руки, вот как хранить мясо, вот как проверять температуру". Конкретные действия, которые можно применить завтра.

**ISO 27002** — это именно такой "учебник" для информационной безопасности. Он не говорит "создайте систему менеджмента" (это ISO 27001). Он говорит: "вот 93 конкретных совета, как защитить информацию".

ISO 27002 — справочник. Его не сертифицируют. Он помогает выбрать и реализовать контроли из ISO 27001.

## Зачем нужен ISO 27002?

### Ситуация 1: Реализация контролей

Компания решила получить ISO 27001. Но как внедрить контроли? ISO 27001 говорит "защитите доступ". А как именно?

ISO 27002 даёт конкретику: "Используйте MFA, настройте роли, проводите пересмотр прав каждые 6 месяцев".

### Ситуация 2: Выбор контролей

У компании ограниченный бюджет. Нужно выбрать, какие контроли внедрить первыми.

ISO 27002 описывает каждый контроль: зачем нужен, как реализовать, что учитывать.

### Ситуация 3: Обучение

Новый сотрудник в команде безопасности. Нужно быстро ввести его в курс дела.

ISO 27002 — структурированный справочник. 93 контроля с описанием.

## Каждый контроль содержит

| Элемент | Описание |
|---------|----------|
| Название | Что это |
| Цель | Зачем нужно |
| Руководство | Как реализовать |
| Дополнительная информация | Ссылки, примеры |

## ISO 27001 vs ISO 27002

| Критерий | ISO 27001 | ISO 27002 |
|----------|-----------|-----------|
| Тип | Стандарт (обязательный) | Справочник (рекомендации) |
| Сертификация | Да | Нет |
| Контролей | 114 (ссылка) | 93 (детали) |
| Фокус | Система менеджмента | Как реализовать |
| Использование | Для аудита | Для внедрения |

Новый контроль в 2022. Рекомендует:
- Собирать информацию об угрозах
- Анализировать актуальность
- Делиться с партнёрами
- Использовать для защиты

### Cloud security

Контроль 5.18 — использование облачных сервисов:
- Оценка рисков облака
- Соглашения с провайдером
- Контроль данных
- Выход из облака

## Ограничения ISO 27002

### Не обязателен

Это справочник. Можно использовать, можно нет. Сертификации нет.

### Общие рекомендации

Контроли описаны общо. Для конкретных технологий — нужны дополнительные руководства.

### Не покрывает всё

Новые технологии (AI, blockchain) — пока нет детальных рекомендаций.

### Ответы на чек-лист

1. **ISO 27002** — справочник по практическим мерам информационной безопасности. Дает конкретные рекомендации по внедрению контролей.

2. **93 контроля** в версии 2022.

3. **4 категории**: Организационные (37), Люди (8), Физические (14), Технологические (34).

4. **ISO 27001** — стандарт системы менеджмента (обязательный для сертификации). **ISO 27002** — справочник с деталями реализации (рекомендации).

5. **Сертификации ISO 27002 нет**. Это справочник, не стандарт для аудита.

6. **Threat intelligence (5.7)** — сбор, анализ и использование информации об угрозах для защиты.

7. **ISO 27002 помогает внедрить ISO 27001**, давая конкретные рекомендации по каждому контролю из Приложения A.

8. **Каждый контроль содержит**: название, цель (зачем), руководство (как), дополнительная информация (ссылки).

9. **Новые в 2022**: threat intelligence (5.7), cloud services (5.18), ICT readiness for business continuity (5.24), consolidated категории.

10. **Ограничения**: не обязателен, общие рекомендации (не для конкретных технологий), не покрывает новейшие технологии.
