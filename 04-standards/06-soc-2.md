# SOC 2: Аудит для SaaS-компаний

## Что такое SOC 2?

Представь, что ты хочешь хранить драгоценности в банке. Ты спрашиваешь: "А безопасно ли у вас?" Банк отвечает: "Да, у нас стальные двери, охрана, сейфы". Ты спрашиваешь: "А кто проверял?" Банк: "Независимый аудитор. Вот его заключение".

**SOC 2** — это именно такое "заключение аудитора" для облачных компаний. Не сертификат, не стандарт — а отчёт. Независимый аудитор проверил: действительно ли компания защищает данные клиентов?

SOC расшифровывается как System and Organization Controls. Разработан AICPA (американский институт CPA). Используется в основном в США, но признаётся глобально.

## Зачем нужен SOC 2?

### Ситуация 1: Крупный клиент

HR-платформа хочет работать с корпорацией. Корпорация спрашивает: "У вас есть SOC 2?" Нет — нет контракта.

SOC 2 — пропуск к Enterprise-клиентам.

### Ситуация 2: Доверие

Пользователи спрашивают: "Мои данные в безопасности?" SOC 2 Type II — доказательство. Аудитор проверял 12 месяцев.

### Ситуация 3: Конкуренция

У конкурента есть SOC 2. У нас — нет. Клиенты уходят к конкуренту.

## Пять Trust Services Criteria

| Критерий | Что проверяет | Пример |
|----------|---------------|--------|
| Security | Защита от несанкционированного доступа | Firewall, MFA, доступ |
| Availability | Доступность системы | Uptime, backup, мониторинг |
| Processing Integrity | Корректность обработки | Валидация, логи, контроль |
| Confidentiality | Конфиденциальность | Шифрование, NDA, доступ |
| Privacy | Приватность данных | Согласие, retention, удаление |

**Важно:** Security — обязательный. Остальные — по выбору.

## Типы отчётов

| Тип | Что проверяет | Срок действия | Глубина |
|-----|---------------|---------------|---------|
| SOC 2 Type I | Дизайн контролей на момент | На дату | Описание |
| SOC 2 Type II | Работа контролей за период | 12 месяцев | Глубокая |

**SOC 2 Type I**: "Контроли спроектированы правильно". Снимок на момент.

**SOC 2 Type II**: "Контроли работали правильно 12 месяцев". Намного ценнее.

## Что проверяет аудитор

| Область | Что смотрит |
|---------|-------------|
| Управление доступом | MFA, роли, отзыв |
| Изменения | Change management |
| Операции | Backup, мониторинг, incidents |
| Риски | Risk assessment |
| Коммуникации | Политики, обучение |
| Поставщики | Vendor management |

## SOC 2 vs ISO 27001

| Критерий | SOC 2 | ISO 27001 |
|----------|-------|-----------|
| Тип | Аудиторский отчёт | Сертификат |
| Фокус | SaaS, облако | Любая организация |
| География | США (глобально) | Мировая |
| Срок | Type I: на дату; Type II: 12 мес | 3 года |
| Контролей | 64-80 (Security) | 114 |
| Дороже? | Обычно да | Обычно нет |
| Ценность | Для клиентов США | Для международных |

## Процесс получения SOC 2

### Шаг 1: Подготовка (3-6 месяцев)

- Создать политики
- Настроить контроли
- Внедрить процессы
- Документировать

### Шаг 2: Readiness assessment

Консультант проверяет: готовы ли к аудиту?

### Шаг 3: Аудит Type I (1-2 месяца)

Аудитор проверяет дизайн контролей.

### Шаг 4: Аудит Type II (6-12 месяцев)

Аудитор наблюдает за работой контролей.

### Шаг 5: Отчёт

Получаем отчёт с мнением аудитора.

## Ограничения SOC 2

### Дорогой

$50,000 - $200,000+ в год. Плюс подготовка.

### Долгий

12+ месяцев для Type II.

### Не стандарт

Это отчёт, не сертификат. Нет логотипа "SOC 2 certified".

### Ответы на чек-лист

1. **SOC 2** — аудиторский отчёт для облачных компаний. Проверяет, защищают ли они данные клиентов.

2. **Пять критериев**: Security (безопасность), Availability (доступность), Processing Integrity (целостность обработки), Confidentiality (конфиденциальность), Privacy (приватность).

3. **Type I** — дизайн контролей на момент. **Type II** — работа контролей за период (обычно 12 месяцев).

4. **Security** — обязательный критерий. Остальные — по выбору.

5. **SOC 2** — аудиторский отчёт для SaaS (США). **ISO 27001** — международный сертификат системы менеджмента.

6. **AICPA** — американский институт certified public accountants.

7. **$50,000 - $200,000+** в год (зависит от размера, выбранных критериев, типа).

8. **Type I** — 1-2 месяца. **Type II** — 12+ месяцев.

9. **Readiness assessment** — проверка готовности к аудиту перед началом.

10. **Ограничения**: дорогой, долгий, не стандарт (отчёт, не сертификат), фокус на SaaS.
