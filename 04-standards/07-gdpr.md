# GDPR и 152-ФЗ: Защита персональных данных

## Что такое GDPR и 152-ФЗ?

Представь дневник. Твой личный дневник, куда ты записываешь мысли, чувства, секреты. Ты хранишь его в ящике стола. Никому не показываешь. Это твоё личное пространство.

Теперь представь, что кто-то взял твой дневник и начал читать. Или продал соседу. Или повесил на стену в школе. Как бы ты себя чувствовал?

**GDPR** (General Data Protection Regulation) — это европейский закон, который говорит: "Дневник каждого человека — это его собственность. Никто не может читать его без разрешения. И если ты хранишь дневники других людей — ты отвечаешь за их сохранность".

**152-ФЗ** (Федеральный закон о персональных данных) — это российский аналог. Он говорит то же самое, но по-русски и с российскими штрафами.

## Зачем нужны GDPR и 152-ФЗ?

### Ситуация 1: HR-платформа хранит резюме

В резюме — имя, телефон, адрес, опыт, зарплата. Это всё персональные данные. HR-платформа ОБЯЗАНА защищать их. Иначе — штраф.

### Ситуация 2: Утечка данных

У компании утекла база клиентов. GDPR требует сообщить о взломе в течение 72 часов. 152-ФЗ — тоже требует уведомлять Роскомнадзор.

### Ситуация 3: Пользователь хочет удалить данные

Кандидат написал: "Удалите моё резюме". Компания ОБЯЗАНА удалить. GDPR — право на забвение. 152-ФЗ — право субъекта на удаление.

## GDPR: ключевые принципы

| Принцип | Описание | Пример |
|---------|----------|--------|
| Законность | Обрабатывать можно только законно | Согласие пользователя |
| Честность | Не обманывать, зачем собираем | Ясная privacy policy |
| Прозрачность | Пользователь знает, что происходит | Информирование |
| Ограничение цели | Собрали для одного — не используем для другого | Резюме для найма, не для рекламы |
| Минимизация | Собираем минимум необходимого | Не спрашиваем лишнее |
| Точность | Данные должны быть актуальными | Обновление резюме |
| Ограничение хранения | Храним не дольше нужного | Удаляем через 2 года |
| Конфиденциальность | Защищаем от утечек | Шифрование, доступ |
| Подотчётность | Можем доказать соблюдение | Аудит, логи |

## Права субъекта (GDPR)

| Право | Описание |
|-------|----------|
| Доступ | Узнать, какие данные хранятся |
| Исправление | Исправить неверные данные |
| Удаление (право на забвение) | Удалить данные |
| Ограничение обработки | Остановить обработку |
| Переносимость | Получить данные в машиночитаемом формате |
| Возражение | Отказаться от обработки |
| Автоматизированные решения | Не быть объектом автоматического профилирования |

## 152-ФЗ: ключевые требования

| Требование | Описание |
|------------|----------|
| Согласие | Нужно согласие на обработку |
| Уведомление | Уведомить Роскомнадзор |
| Категории | Особые категории (раса, религия, здоровье) — усиленная защита |
| Трансграничная передача | Данные в другие страны — только с разрешения |
| Ответственный | Назначить ответственного за ПДн |
| Разрушение | Уничтожить при достижении цели |
| Безопасность | Защитные меры (организационные и технические) |

## Штрафы

| Закон | Максимальный штраф | За что |
|-------|---------------------|--------|
| GDPR | €20 млн или 4% оборота | Серьёзные нарушения |
| GDPR | €10 млн или 2% оборота | Несерьёзные |
| 152-ФЗ | 500 млн руб. | Значительные нарушения |
| 152-ФЗ | 75 млн руб. | Незначительные |

## GDPR vs 152-ФЗ

| Критерий | GDPR | 152-ФЗ |
|----------|------|--------|
| География | ЕС | Россия |
| Экстерриториальность | Да (если обрабатываете данных ЕС) | Да (если обрабатываете в РФ) |
| Штрафы | До €20 млн | До 500 млн руб. |
| Согласие | Явное | Упрощённое (можно молчаливое) |
| DPO | Обязан при >250 сотрудников | При >100 сотрудников |
| Уведомление о breach | 72 часа | Немедленно |
| Право на забвение | Да | Есть ограничения |

## Архитектурные решения

### Privacy by Design

Безопасность закладывается на этапе проектирования. Не добавляется потом.

### Data Minimization

Собираем минимум. Не спрашиваем "на всякий случай".

### Consent Management

Платформа для управления согласиями. Пользователь может дать, отозвать, проверить.

## Ограничения

### Конфликт законов

GDPR требует удалить. 152-ФЗ требует хранить для налоговой. Конфликт.

### Трансграничная передача

Передать данные из России в ЕС — сложно. И наоборот.

### Субъективность

"Легитимный интерес" vs "согласие". Интерпретация зависит от регулятора.

### Ответы на чек-лист

1. **GDPR** — европейский регламент о защите персональных данных. Применяется к данным резидентов ЕС.

2. **152-ФЗ** — российский федеральный закон о персональных данных.

3. **9 принципов GDPR**: законность, честность, прозрачность, ограничение цели, минимизация, точность, ограничение хранения, конфиденциальность, подотчётность.

4. **Права субъекта**: доступ, исправление, удаление (право на забвение), ограничение обработки, переносимость, возражение, не быть объектом автоматического профилирования.

5. **Штрафы GDPR**: до €20 млн или 4% оборота (серьёзные), до €10 млн или 2% (несерьёзные).

6. **Штрафы 152-ФЗ**: до 500 млн руб. (значительные), до 75 млн (незначительные).

7. **Отличия**: GDPR — ЕС, 152-ФЗ — Россия. GDPR требует явного согласия, 152-ФЗ допускает молчаливое. GDPR — DPO при 250+, 152-ФЗ — при 100+.

8. **Privacy by Design** — безопасность закладывается на этапе проектирования системы.

9. **DPO** — Data Protection Officer. Специалист по защите данных. Обязателен при определённых условиях.

10. **Ограничения**: конфликт законов (удаление vs хранение), трансграничная передача, субъективность интерпретации.
